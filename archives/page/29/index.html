Ruby写代码须知



新手须知外，写代码除了有个编辑器外，考虑多的就是怎么调试代码。

Ruby Debugger，Ruby本身提供了调试功能，可以使用-r debug选项，调试时的命令参见When Trouble Strikes，我看也gdb有点像，break, where, list, cont。

当然也肯定离不开irb，交互式Ruby。不过现在有个两者的结合体，又可以象irb一样交互，又提供调试功能，那就是pry，这里有篇简单的入门文章，看起来不错，后续在实践中熟悉。pry-nav提供了上一步、下一步的功能。

还有两个性能方面的分析工具，一个是benchmark，分析整体消耗的时间，一个是profier，能分析到每个方法的执行结果数据，就能方便定位性能瓶颈了。

Ruby新手须知



有鉴于写代码老是犯低级错误，先看一下新手须知。原帖在Things That Newcomers to Ruby Should Know ，选了几个我感兴趣和看明白的。

1. 使用”ruby -w”能得到更有帮助警告，可以设置RUBYOPT环境变量为’w’。
2. ri 查看Ruby文档。
3. 注意hash，如果key值的对象变化的，需要Hash#rehash。

s = “mutable”
arr = [s]
hsh = { arr => “object” }
s.upcase!
p hsh[arr] # -> nil (maybe not what was expected)
hsh.rehash
p hsh[arr] # -> “object”
</pre> 4. 从文件中读到的数据是string类型，需要to_i, to_f转换为数值，否则+做的是字符串连接。 5. ruby没有++，--，需要使用x+=1。 6. ruby有两类逻辑运算符[!, &amp;&amp;, ||] 和 [not, and, or]，牵着的优先级比赋值运算符高，而后者比赋值运算符低。并且注意&amp;&amp;比||优先级高，但and和or是一样的。这会有有趣的现象： <pre>
a = ‘test’
b = nil
both = a && b # both == nil
both = a and b # both == ‘test’
both = (a and b) # both == nil
</pre> 7. case语句调用的是===方法，而不是==。 8. 建议不要在方法调用'('加上空格。 9. "0..k"表示的是一个Range对象，所以从0到k的循环用的是(0..k).each，而不是[0..k].each。 10. 只有false和mil才表示false，0,"",[],{}都bei被认为是true。 11. Ruby变量保存是对象引用，赋值拷贝的是引用。举例： <pre>
a = ‘aString’
c = a
a += ‘ modified using +=’
puts c # -> “aString”
a = ‘aString’
c = a
a << ‘ modified using <<’
puts c # -> “aString modified using <<”

12. 还是引用的问题，没有built-in copy标准。一种方式是采用serialization/marshalling，序列化和反序列化达到类似效果。

Ruby多进程



Ruby有很多调用和管理独立进程的方法。

### system和反引号最简单的生成一个独立的进程的方式是运行一些命令，然后等它完成。可以采用system和反引号方法，比如：

system(“ls”)
result = date
result
</pre> Kernel::system在子进程中执行指定的命令，如果命令能被找到并执行返回true，否则返回false，子进程的返回状态放在全局变量$?中。 system的问题是命令的输出结果和程序的输出混在一起，可能不是你所预期。而要获得子进程的标准输出，可以使用反引号，记得返回结果使用string.chomp 移去换行字符。 ### popen 有时我们需要多一点的控制，比如和子进程进行通信，传个数据啥的。IO.popen可以实现，在子进程运行命令，并且将子进程的标准输入和输出连接到Ruby IO对象。 而在popen的参数为"-"时，popen会fork一个新的Ruby解释器，这个解释器和原先的解释器在popen调用返回后都继续运行，原先的进程接收到一个IO对象，而子进程则接收nil，比如： <pre>
pipe = IO.popen(“-“,”w+”)
if pipe
pipe.puts “Get a job!”
$stderr.puts “Child #{Process.pid} says ‘#{pipe.gets.chomp}’”
else
$stderr.puts “Dad #{Process.pid} says ‘#{gets.chomp}’”
puts “OK”
end
</pre> 相对于popen，传统的Unix采用Kernel::fork, Kernel::exe和IO.pipe方式也被支持。 ### 独立的子进程 fork 在父进程中返回进程编号，在子进程中返回nil。子进程执行exec，我们可以在父进程中wait。 <pre>
exec(“sort testfile > output.txt”) if fork == nil
# The sort is now running in a child process
# carry on processing in the main program

# then wait for the sort to finish
Process.wait
</pre> 如果你想在子进程退出时接到通知，而不是等，可以使用Kernel.trap 设置信号处理，比如： <pre>
trap(“CLD”) {
pid = Process.wait
puts “Child pid #{pid}: terminated”
exit
}

exec(“sort testfile > output.txt”) if fork == nil

# do other stuff…
</pre> 写循环的时候想当然地写了i++，却怎么都报错，才发现[Ruby原来并不支持++，--运算符](http://stackoverflow.com/questions/3660563/why-doesnt-ruby-support-i-or-i-increment-decrement-operators)。 ### 块和子进程 IO.popen 可以像File.open一样使用块的方式，比如： <pre>
IO.popen (“date”) { |f| puts “Date is #{f.gets}” }
</pre> 这样IO对象会在块代码退出时自动关闭。 而对于Kernel.fork来说，fork后的块在子进程里运行，块后的代码在父进程中运行，这个比判断返回的进程Id要方便多了。 <pre>
fork do
puts “In child, pid = #$$”
exit 99
end
pid = Process.wait
puts “Child terminated, pid = #{pid}, exit code = #{$? >> 8}”

小技巧：$?按位右移8个二进制位，是Posix中表示返回值中低8位是程序退出的原因，高8位是真实的返回代码。

好皇帝和他的坏儿子（二）

马可.奥勒留的原则之一是“宽恕你的敌人”。尽管马可.奥勒留不是基督徒，但他的为人处世比后世那些身为基督徒的皇帝要高尚得多。
很多人能够以身作则，却无法教好自己的儿子，马可.奥勒留也是如此，他没能把自己的儿子也培养成为一个好人。马可可以说是好到了极致，而他的儿子康茂德确是坏到了…
“极致。”小乐插话。
这个孩子长大成人可以自己做主后，他把履行职责、行为端正和遵从上帝的法则统统抛在脑后。第一目标就是享乐，而且是最堕落的享乐。
“我猜对了。”
康茂德是个运动健将，有结实的肌肉和英俊的外表，为此还给自己做了个雕像，让人们把自己当神一样供奉。为了炫耀肌肉的健美和力量，作为一个皇帝却参加摔跤比赛。要是有人挑他的过错，他就会把这个人毒死或杀死。他过着野性、放荡的生活，不过最终也罪有应得，很多刺杀他的计划都没有成功，但最后被一个摔跤手给勒死了。
“啊？！是摔跤吗？”
“不是，应该也是被刺杀的。”
“192年的最后一天，由康茂德的情妇瑪琪亞、近卫军长官萊图斯計劃，在康茂德从竞技场回宮洗浴時，由摔跤手納尔奇苏斯將他勒死在浴場。”

快乐啊快乐！对于康茂德，快乐就意味着尽情地吃喝玩乐，参加通宵达旦的放荡的晚会。而对哪些不同的愉悦，也就是高尚的乐趣，康茂德是没兴趣的。
大约在芝诺的同一时期，还有一位希腊哲学家，叫伊壁鸠鲁。他的思想后来也在罗马流行起来，伊壁鸠鲁主义者认为最高的善就是快乐，但是这种快乐必须是“正确的”（善的）。
念着念着，觉得鸠字我念的不对，我念zhen音，一查，果然，应该是jiu，赶紧向小乐道歉纠正。
“鸠字怎么写的？”小乐问。
“左边一个九，右边一个鸟。”
“那是应该念鸠。”
惭愧，想一想这不是斑鸠的鸠嘛。

看《哲学的故事》中的描述，伊壁鸠鲁主义差不多是思想家伊壁鸠鲁独立创立的，其首要目的在于把人们从恐惧中解放出来，尝到人们去寻求个人生活的幸福和成就。“默默无闻地生活”是其格言之一。

而在希利尔讲世界史中说伊壁鸠鲁主义者认为这些快乐是善的：

  • 为人诚实、正直

  • 和好人交朋友

  • 过简朴的生活

  • 安静地学习

  • 保持镇静
    伊壁鸠鲁主义者认为，任何导致痛苦地快乐都不是真正地快乐——应该说根本鸠不是快乐。想一想，如果康茂德遵循地是伊壁鸠鲁的快乐理念，而不是他自己那种放任、自私的想法，那他应该会多么幸福啊！

Ruby没有重载



写代码的时候想在构造时，传递不同的参数方式，想当然的就是两个initialize，比如：

def initialize(level = 0)
# do something
end

def initialize(nums, opers, isDivided = true)
# do something
end
</pre> 结果运行时直接报错： <pre>
mathstest.rb:11:in `initialize’: wrong number of arguments (1 for 2) (ArgumentError)

原来Ruby并不支持类方法中的重载（overload），参见：Why does Ruby not support method overloading，其中说到：

方法重载有有两种方式，一是参数类型不同；二是参数个数不同。在Ruby中首先就没有数据类型的定义，它本身是一种动态类型的语言。而第二种方式看起来可行，但是参数可以有缺省值，这时调用谁也是个问题。

看书是一回事，写代码是一回事



光看书感觉还不错，但写起代码来有挫折感，小问题不断。记两个小问题：

之前说采用Sublime Text 2来写Ruby代码，并且Command + B可以直接运行查看结果，觉得挺好，但不知为什么就是无法实现输入。比如最简单的：

puts gets
puts STDIN.gets
</pre> 在Sublime Text2运行结果就是打印两空行，毫无提示需要输入。而在控制台运行一切正常。搜索半天无果，后来就自己到[stackoverflow](http://stackoverflow.com)上提了[一个问题](http://stackoverflow.com/questions/17437814/ruby-stdin-gets-return-nil-directly-while-running-in-sublime-text2)（这还是我第一次）。 得到的回复是Sublime并不支持在bulid下输入，很重要一点是Sublime本身不是一个IDE，而只是一个强大的文本编辑工具，还是需要配合IRB、调试器来运行跟踪程序。看来离美好还差一点点。 还有个问题，更简单，直接从ProgrammingRuby上复制了段[多线程代码](http://www.ruby-doc.org/docs/ProgrammingRuby/html/tut_threads.html)，涉及http请求，如： <pre>
resp, data = h.get(‘/‘, nil )
</pre> 但运行时就报错： <pre>
/Users/yangbin/.rvm/rubies/ruby-1.9.3-p125/lib/ruby/1.9.1/net/http.rb:1019:in get': undefined methodkeys’ for nil:NilClass (NoMethodError)
</pre> 还是版本问题，可以看到Net::HTTP.get 在[1.9.2](http://rubydoc.info/stdlib/net/1.9.2/Net/HTTP:get) 和[1.8.6](http://rubydoc.info/stdlib/net/1.8.6/Net/HTTP.get) 的版本上参数发生了变化。 <pre>
- (Object) get(path, initheader = {}, dest = nil, &block) #1.9.2
+ (Object) get(uri_or_host, path = nil, port = nil) #1.8.6

代码get参数中传递的第二个参数nil含义完全不同了，简单去掉nil即可。

看书是一回事，写代码是一回事，还得练兵才是。

BTW. 在stackoverflow提问题的好处，还有人会帮助你把问题的书写英文给改写了一遍，可以练习英语。

好皇帝和他的坏儿子（一）

“好皇帝和他的坏儿子”，小乐选的，选完之后她说：“这在中国就是说不肖子孙了。”

在邪恶的罗马皇帝尼禄死了一百多年后，有个名叫马可.奥勒留的新皇帝登上了皇位。与尼禄的邪恶正好相反，他非常善良。

“尼禄你知道的哦？”

“嗯。”

在这个时候，大多数人对任何宗教都缺乏信仰。他们既不是基督徒，对本国供奉的朱庇特、朱诺和其他神灵也谈不上多么虔诚。

“罗马是从哪个皇帝开始信奉基督教的？”（重复也是一种好的学习方式。）

“君士坦丁，梦见十字架然后打了胜战。”（看来之前讲的还没忘记。）

小乐又说：“朱诺的名字，我觉得很怪的，有点象猪。”

“罗马的神是和古希腊的神一样的。”

“我在心里是把罗马和希腊当作差不多的。”小乐说。

“罗马的文化是从希腊传来的。”

“希腊是罗马的老师。”

 

闲扯开来后，小乐又中途给我插一段广告：

我用am，你用are，

is前面他，她，它，

男他，女她，宝盖它。

“Miss He教的？”

“不是，Mr Lin。”

 

广告之后，继续回来。

 

大约在公元前300年，希腊哲学家芝诺讲授一种称为“斯多葛主义”的哲学，他的思想在一个世纪后传到了罗马。又过了一百年，马可.奥勒留皇帝出现了，他也是个斯多葛主义者。他还把自己的思想记录下来，就是《沉思录》。

“你的漫画哲学里有没有奥勒留的？”

“我不知道，我没看完。”

“让老爸来找找。”

我翻出了《漫画哲学》和《哲学的故事》，两本书中都提到了斯多葛主义和马可.奥勒留。

关于斯多葛主义的由来，是因为希腊的大型建筑物多被很多柱子围起来，走廊通常在柱子和墙壁之间，这种走廊被称为柱廊，柱廊在希腊语中是stoa，斯多葛主义的名称就来源于此。

斯多葛学派的核心观点就是，没有什么权威比理性更高。像马可.奥勒留所信奉的一些思想：

  • 我必须平静的承受痛苦和伤害。
  • 我必须一贯履行自己的职责。
  • 我不可一味追求享乐。
  • 行为端正是生活中最好的事情。
  • 我必须遵从上帝的法则。
    等等。

 

直到今天，成千上万的人们还在阅读马可.奥勒留的《沉思录》，书中的一些言论听起来就像是出自圣经一样。甚至到了今天，人们还把那些承受苦难而毫无怨言的人称作斯多葛主义者。

数字游戏抢30

周六下午好不容易天气放晴，晚上就又出去轮滑了。路上，小乐和乐妈在玩抢50的游戏，游戏规则是两人交替数数，可以说1～3个数字，最多3个，看谁先抢到50。

两人抢地不亦乐乎，中间偶尔思考一番，不过还是错进错出，关键数字老丢。

我就问小乐：“抢50的关键是什么？”

小乐：“抢5的倍数。”

哦，难怪，小家伙还是当成抢30在玩。

抢30以前和小乐玩过，非常不错的一个游戏，游戏中让孩子对数字产生兴趣。基本规则是每人只能说1～2个数字，抢到30的为胜。估计这个抢50也是她们俩从抢30中变化而来。

记得自己小时候也喜欢玩这个游戏，一开始总是输，不管是先说还是后说，到后来发现诀窍，也就是要抢3的倍数，让对方先说就可以了。游戏也就变得无味，只能一直保存在心底，直到可以和自己的孩子开始玩。于是孩子和你又重复着以前的故事。

乐乐也明白了要抢3的倍数，别人说一个，你就说两个，别人说两个，你就说一个，就是这么简单。

那么回到抢50，是怎么回事呢？为什么变成抢5的倍数也不对呢？不急着公布答案。

“小乐，你要抢到50的话，前面那个数字你必须要抢到什么？”

小乐陷入了思考，然后回答：“46。”

“为什么呢？”

“我说46，对方就说不到50，我就能抢到了。”

“嗯，那再之前呢？”

“42。”这次明显速度加快。

“再之前呢？”

“38。”

此处省略n次交互。

“那你发现有什么规律了吗？”

“每一次都要减4。”

“好的，那么第一个数字要抢什么？”

“嗯…要抢2。”

“对了，那你是要先说，还是后说？”

“先说。”

于是小乐和我PK了一次，我毫无疑问地输了。

还没完。

“2是怎么算出来的？”

“50除以4，余数是2。”

这就是学过除法和余数的孩子，就不一样了。实际上小乐她们的数学题目中类似日期的题目，今天星期几，过了一年后是星期几就是这样的题目。

“那么为什么是除以4呢？”

除以的数字，和要抢的数字没有关系，比如说抢50，并不是除以5。而是因为我们每个人最多只能说3个数字，两个人轮流说的数字是以4为周期，所以抢到第一个数字后，接下来跟着别人就可以了，别人说1个，你就说3个，别人说2个，你也说2个，别人说3个，你就说1个。

“明白了吗？”

“明白了。”

还没完。

我又继续问：“那我们现在改变一下游戏规则，谁抢到50谁输？你是要先抢还是后抢？”

这个问题有点绕，过程就不继续说了，有兴趣的家长朋友可以和孩子玩一玩。

BTW.24点也是一个古老的数字游戏，两年前讲过陪小乐玩24点的游戏，从幼儿园开始我们就看车牌开始玩一直玩到现在也乐此不疲。

Ruby基本的输入输出



Ruby提供了两种I/O方式：

一种是Kernel模块中实现的I/O相关的方法，gets, open, print, printf, putc, puts, readline, readlines和test，提供简单和方便的输入输出操作。 一种是使用IO对象，让你有更多的控制。

IO对象有两个子类File和BasicSocket提供具体的行为，但基本原则是相同的。一个IO对象在Ruby程序和外部资源之间建立了双向通道（读、写）。

Ruby可以使用File.open打开文件，并传递一个块，执行完成后自动关闭文件。File.open如果不传递block，则和File.new本质是相同。

File.open(“testfile”, “r”) do |aFile|

# … process the file

end
</pre> 读取文件的方式 * IO.each_byte和IO.each_line分别提供了逐个字符和逐行读取文件的方式，each_line还可以设定自定义的行分隔符，each等同于each_line。 <pre>
IO.foreach(“testfile”){ |line| puts line}

IO.for_each提供了iteraotr和自动关闭文件的功能。 IO.readlines可以将文件直接读取到行数组中。

写文件，puts和print一个对象，会把对象采用to_s转成字符串，如果to_s没有返回有效的字符串，则会创建诸如ClassName:0x123456形式的对象类名和id的字符串。

IO.sysread和IO.syswrite提供了更底层的读写能力。 Array.pack, String.unpack提供了字符串和二进制流之间的打包和解包能力。
还可以往输出IO流使用<<运算符，*注意往String使用<<时，对于0到255的整型数值会转换成字符再连接。

网络交互提供两种方式，一种是较低层次的socket通信，一种是较高层次的应用层通信，比如FTP, HTTP, POP, SMTP和telnet。
