看书是一回事，写代码是一回事



光看书感觉还不错，但写起代码来有挫折感，小问题不断。记两个小问题：

之前说采用Sublime Text 2来写Ruby代码，并且Command + B可以直接运行查看结果，觉得挺好，但不知为什么就是无法实现输入。比如最简单的：

puts gets
puts STDIN.gets
</pre> 在Sublime Text2运行结果就是打印两空行，毫无提示需要输入。而在控制台运行一切正常。搜索半天无果，后来就自己到[stackoverflow](http://stackoverflow.com)上提了[一个问题](http://stackoverflow.com/questions/17437814/ruby-stdin-gets-return-nil-directly-while-running-in-sublime-text2)（这还是我第一次）。 得到的回复是Sublime并不支持在bulid下输入，很重要一点是Sublime本身不是一个IDE，而只是一个强大的文本编辑工具，还是需要配合IRB、调试器来运行跟踪程序。看来离美好还差一点点。 还有个问题，更简单，直接从ProgrammingRuby上复制了段[多线程代码](http://www.ruby-doc.org/docs/ProgrammingRuby/html/tut_threads.html)，涉及http请求，如： <pre>
resp, data = h.get(‘/‘, nil )
</pre> 但运行时就报错： <pre>
/Users/yangbin/.rvm/rubies/ruby-1.9.3-p125/lib/ruby/1.9.1/net/http.rb:1019:in get': undefined methodkeys’ for nil:NilClass (NoMethodError)
</pre> 还是版本问题，可以看到Net::HTTP.get 在[1.9.2](http://rubydoc.info/stdlib/net/1.9.2/Net/HTTP:get) 和[1.8.6](http://rubydoc.info/stdlib/net/1.8.6/Net/HTTP.get) 的版本上参数发生了变化。 <pre>
- (Object) get(path, initheader = {}, dest = nil, &block) #1.9.2
+ (Object) get(uri_or_host, path = nil, port = nil) #1.8.6

代码get参数中传递的第二个参数nil含义完全不同了，简单去掉nil即可。

看书是一回事，写代码是一回事，还得练兵才是。

BTW. 在stackoverflow提问题的好处，还有人会帮助你把问题的书写英文给改写了一遍，可以练习英语。

好皇帝和他的坏儿子（一）

“好皇帝和他的坏儿子”，小乐选的，选完之后她说：“这在中国就是说不肖子孙了。”

在邪恶的罗马皇帝尼禄死了一百多年后，有个名叫马可.奥勒留的新皇帝登上了皇位。与尼禄的邪恶正好相反，他非常善良。

“尼禄你知道的哦？”

“嗯。”

在这个时候，大多数人对任何宗教都缺乏信仰。他们既不是基督徒，对本国供奉的朱庇特、朱诺和其他神灵也谈不上多么虔诚。

“罗马是从哪个皇帝开始信奉基督教的？”（重复也是一种好的学习方式。）

“君士坦丁，梦见十字架然后打了胜战。”（看来之前讲的还没忘记。）

小乐又说：“朱诺的名字，我觉得很怪的，有点象猪。”

“罗马的神是和古希腊的神一样的。”

“我在心里是把罗马和希腊当作差不多的。”小乐说。

“罗马的文化是从希腊传来的。”

“希腊是罗马的老师。”

 

闲扯开来后，小乐又中途给我插一段广告：

我用am，你用are，

is前面他，她，它，

男他，女她，宝盖它。

“Miss He教的？”

“不是，Mr Lin。”

 

广告之后，继续回来。

 

大约在公元前300年，希腊哲学家芝诺讲授一种称为“斯多葛主义”的哲学，他的思想在一个世纪后传到了罗马。又过了一百年，马可.奥勒留皇帝出现了，他也是个斯多葛主义者。他还把自己的思想记录下来，就是《沉思录》。

“你的漫画哲学里有没有奥勒留的？”

“我不知道，我没看完。”

“让老爸来找找。”

我翻出了《漫画哲学》和《哲学的故事》，两本书中都提到了斯多葛主义和马可.奥勒留。

关于斯多葛主义的由来，是因为希腊的大型建筑物多被很多柱子围起来，走廊通常在柱子和墙壁之间，这种走廊被称为柱廊，柱廊在希腊语中是stoa，斯多葛主义的名称就来源于此。

斯多葛学派的核心观点就是，没有什么权威比理性更高。像马可.奥勒留所信奉的一些思想：

  • 我必须平静的承受痛苦和伤害。
  • 我必须一贯履行自己的职责。
  • 我不可一味追求享乐。
  • 行为端正是生活中最好的事情。
  • 我必须遵从上帝的法则。
    等等。

 

直到今天，成千上万的人们还在阅读马可.奥勒留的《沉思录》，书中的一些言论听起来就像是出自圣经一样。甚至到了今天，人们还把那些承受苦难而毫无怨言的人称作斯多葛主义者。

 

题图为马可.奥勒留。

数字游戏抢30

周六下午好不容易天气放晴，晚上就又出去轮滑了。路上，小乐和乐妈在玩抢50的游戏，游戏规则是两人交替数数，可以说1～3个数字，最多3个，看谁先抢到50。

两人抢地不亦乐乎，中间偶尔思考一番，不过还是错进错出，关键数字老丢。

我就问小乐：“抢50的关键是什么？”

小乐：“抢5的倍数。”

哦，难怪，小家伙还是当成抢30在玩。

抢30以前和小乐玩过，非常不错的一个游戏，游戏中让孩子对数字产生兴趣。基本规则是每人只能说1～2个数字，抢到30的为胜。估计这个抢50也是她们俩从抢30中变化而来。

记得自己小时候也喜欢玩这个游戏，一开始总是输，不管是先说还是后说，到后来发现诀窍，也就是要抢3的倍数，让对方先说就可以了。游戏也就变得无味，只能一直保存在心底，直到可以和自己的孩子开始玩。于是孩子和你又重复着以前的故事。

乐乐也明白了要抢3的倍数，别人说一个，你就说两个，别人说两个，你就说一个，就是这么简单。

那么回到抢50，是怎么回事呢？为什么变成抢5的倍数也不对呢？不急着公布答案。

“小乐，你要抢到50的话，前面那个数字你必须要抢到什么？”

小乐陷入了思考，然后回答：“46。”

“为什么呢？”

“我说46，对方就说不到50，我就能抢到了。”

“嗯，那再之前呢？”

“42。”这次明显速度加快。

“再之前呢？”

“38。”

此处省略n次交互。

“那你发现有什么规律了吗？”

“每一次都要减4。”

“好的，那么第一个数字要抢什么？”

“嗯…要抢2。”

“对了，那你是要先说，还是后说？”

“先说。”

于是小乐和我PK了一次，我毫无疑问地输了。

还没完。

“2是怎么算出来的？”

“50除以4，余数是2。”

这就是学过除法和余数的孩子，就不一样了。实际上小乐她们的数学题目中类似日期的题目，今天星期几，过了一年后是星期几就是这样的题目。

“那么为什么是除以4呢？”

除以的数字，和要抢的数字没有关系，比如说抢50，并不是除以5。而是因为我们每个人最多只能说3个数字，两个人轮流说的数字是以4为周期，所以抢到第一个数字后，接下来跟着别人就可以了，别人说1个，你就说3个，别人说2个，你也说2个，别人说3个，你就说1个。

“明白了吗？”

“明白了。”

还没完。

我又继续问：“那我们现在改变一下游戏规则，谁抢到50谁输？你是要先抢还是后抢？”

这个问题有点绕，过程就不继续说了，有兴趣的家长朋友可以和孩子玩一玩。

BTW.24点也是一个古老的数字游戏，两年前讲过陪小乐玩24点的游戏，从幼儿园开始我们就看车牌开始玩一直玩到现在也乐此不疲。

Ruby基本的输入输出



Ruby提供了两种I/O方式：

一种是Kernel模块中实现的I/O相关的方法，gets, open, print, printf, putc, puts, readline, readlines和test，提供简单和方便的输入输出操作。 一种是使用IO对象，让你有更多的控制。

IO对象有两个子类File和BasicSocket提供具体的行为，但基本原则是相同的。一个IO对象在Ruby程序和外部资源之间建立了双向通道（读、写）。

Ruby可以使用File.open打开文件，并传递一个块，执行完成后自动关闭文件。File.open如果不传递block，则和File.new本质是相同。

File.open(“testfile”, “r”) do |aFile|

# … process the file

end
</pre> 读取文件的方式 * IO.each_byte和IO.each_line分别提供了逐个字符和逐行读取文件的方式，each_line还可以设定自定义的行分隔符，each等同于each_line。 <pre>
IO.foreach(“testfile”){ |line| puts line}

IO.for_each提供了iteraotr和自动关闭文件的功能。 IO.readlines可以将文件直接读取到行数组中。

写文件，puts和print一个对象，会把对象采用to_s转成字符串，如果to_s没有返回有效的字符串，则会创建诸如ClassName:0x123456形式的对象类名和id的字符串。

IO.sysread和IO.syswrite提供了更底层的读写能力。 Array.pack, String.unpack提供了字符串和二进制流之间的打包和解包能力。
还可以往输出IO流使用<<运算符，*注意往String使用<<时，对于0到255的整型数值会转换成字符再连接。

网络交互提供两种方式，一种是较低层次的socket通信，一种是较高层次的应用层通信，比如FTP, HTTP, POP, SMTP和telnet。
