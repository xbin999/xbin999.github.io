Ruby的表达式和赋值



Ruby中所有的东西都是一个表达式。 许多运算符其实是方法调用，比如ab+c，实际上是(a.(b)).+(c)，通过a请求对象引用调用方法”“，传递一个参数b，再把结果作为一个对象调用方法”+”，传递c作为参数。由于都是对象，所以可以重定义方法，比如：

class Fixnum
alias oldPlus +
def +(other)
oldPlus(other).succ
end
end

暑假倒计时

马上快期末考试了，最近小乐的作业不是一般地多。原先我要不加班回到家，她大多都已经完成作业，一起吃好晚饭，就可以出去玩了（最近是轮滑时间）。可最近作业都要做3、4个小时，中间吃个饭，做完作业都是要到8点多了。

套用我们家常用文明骂人话：“TMDNNDX！”
这作业也太多了，不就是个期末考试，至于嘛！

前两天，我第一次听到小乐抱怨有关作业的事情，觉得做得好烦，每天都有那么多作业，连玩的时间都没有了。这就是好好的把一件有趣的事情往死里整，把你整厌烦了为止。

说归说，怎么办呢？
有时在家里会说，要不我们去和老师说说：“申请作业免做吧？”虽然有看到有文章提到有类似做法，但觉得有难度，怕老师有看法。（姑且把这方法留着）
那我们就把每天的玩坚决地保留，不管作业做地多晚，玩是必须的，想想穿上轮滑，在夜风中畅快地滑行该是何等地惬意。
那又如何看待这最后的几天呢？坚持一下？

不。
前天晚上，小乐忙了几个小时的作业，一起吃晚饭。
我也不和小乐说考试的事，直接问：“小乐，快放假了吧？”
“嗯。”小乐有点无精打采。
“那我们来个倒计时吧。还有几天啊？”
“下周四考试，考完后，后面就只要领成绩单了。”
“那我们周五就放假了，来算算。”
我一边勾手指头一边算：“周四1天，周五2天，周六3天，周日4天，一5…”
这一一数来的感觉无疑是快乐的，小乐也加入了。
“二6，三7，四8，周五就放假了。”
“那就只有8天，我们在板上写下来，然后每天改一次。”
“好。”
“中间还有一个礼拜六，礼拜天，也是休息的，啊呀，那就相当于只有6天了。”
“是呀。”小乐心情大好。

今天一早，躺在床上的时候，小乐和我说：“爸爸，只有6天了。”
“什么只有6天了？”
“放暑假呀！”
“啊，对呀，你改倒计时了吗？”
“改了。”

换一个角度看问题，生活可以更美好，不是吗？

有关Ruby的方法



1. 用def定义，小写字母开头，只有方法(method)，有返回值。
2. 以?结尾的方法表示是否的判断。
3. 以!结尾的方法会修改方法的接收者(receiver)，比如chop和chop!。
4. 用return返回，如果没有return则返回最后一个表达式的值。
5. 可以使用*变量名，作为可变参数传递任意多个参数，传递的多个参数以数组形式保存在指定的参数名中。
6. 反过来方法有多个参数，调用时也可以传递一个数组，依次赋给各变量。
7. 可以传递block，通常用yield调用，也可以把block保存在变量中，会转换为Proc对象。
8. 典型的调用形式：接收者，方法名，参数和块(block)，如果不指定接收者，缺省为self。如：

connection.downloadMP3(“jitterbug”) { |p| showProgress(p) }

正则表达式中的符号



最基本的有关正则表达式的使用前面说了，有关Regexp类中的$符号也有说明，再补充说明一下一些常用的符号。

正则表达式中有几部分内容：

1. 先是patterns，所有字符除了., |, (, ), [, {, +, \, ^, $, , 和 ?之外，都匹配自己本身。接下来其实就是了解这些乱七八糟的符号分别干嘛的。\ 用来做字符转义。
2. anchors，^, $是最基本的表示首和尾，除此之外还有\A, \z, \Z, \b, \B。
>- \A 字符串的开始
>- \z 字符串的结尾
>- \Z 支持以换行符结束的字符串的结尾
>- \b 单词首尾
>- \B 非单词首尾

1.9.3p125 :024 > showRE(“this is the time”, /time\z/)

`
3. 字符类(character class)，放在[]中，有\d, \D, \s, \S, \w, \W。大写字母表示否定。
>- \d [0-9]数字
>- \D [^0-9]非数字
>- \s [\s\t\r\n\f] 空格
>- \S [^\s\t\r\n\f] 非空格 >- \w [A-Za-z0-9] 单词字符
>- \W[^A-Za-z0-9] 非单词字符4. 用于表示重复，, +, ?，还有{m,n}, {m,}的形式。
5. 替换，|
6. 组合，()

乱七八糟的字符就说完了，还有一个常用的是替换，也就是sub和gsub，这里会用到\1, \2等模式中匹配到的变量，比如：

`

Ruby中正则表达式的类处理



正则表达式的技巧太多了，还是需要实际经验，积累一些技巧。

不过最让我看到有兴趣的还是Ruby中正则表达式也是对象，只不过再基于“$特殊变量”封装使得所有的处理看起来和Perl的风格处理类似，挺有趣。

看下面的两个例子，能很好的说明。

用//括起来的正则表达式是Regexp对象，match匹配的结果是一个MatchData实例，MatchData是一个特殊变量$~的类型，封装了通过特殊变量$&,$’, $, $1, $2等等返回的结果。 re = /(\d+):(\d+)/ # match a time hh:mm re.class # Regexp md = re.match("Time: 12:34am") md.type » MatchData md[0] # == $&amp; » "12:34" md[1] # == $1 » "12" md[2] # == $2 » "34" md.pre_match # == $ » “Time: “
md.post_match # == $’ » “am”
</pre> 看明白了没？那么如果做连续两次正则表达式匹配，特殊变量$存放的会是最后一次的结果，这时你可以把上一次的匹配结果MatchData用变量保存起来，需要的时候恢复即可。 <pre>
re = /(\d+):(\d+)/
md1 = re.match(“Time: 12:34am”)
md2 = re.match(“Time: 10:30pm”)
[ $1, $2 ] # last successful match » [“10”, “30”]
$~ = md1
[ $1, $2 ] # previous successful match » [“12”, “34”]
