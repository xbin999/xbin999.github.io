我卖掉的第一份货物

今天是求是教育集团一年一度的贸易节，也是全校所有同学最喜爱的节日之一。

吃完中饭，走廊上立刻“火”了。课桌和椅子相互碰撞，道路上人山人海，连一只小猫都钻不出去。我和刘智成、戴欣月，推着桌椅艰难的，以乌龟的速度向三年级办公室挪去。经过一番努力后，我们终于到达了目的地，摆好了摊。

我拿着一个猴子橡皮和一个香蕉橡皮去卖。走到三（4）班门口时，看到徐安淇正坐在摊位上，我知道她是一名橡皮爱好者。我走到她面前，学着卖木雕的少年那课中黑人少年诚恳地说：“安淇，你买一个吧！”没想到她对我说：“啊，芷欣！我身无分文，我老爸正在给我宅急送的路上呢！”“Uh Oh！”我吃惊地叫了一声，又转向在徐安淇身旁的施蕴嘉，“买一个吧！”我对她说。她犹豫了一会儿，问我：“两块加起来几元？”我回答说：“三元。”她从皮夹里拿出三元钱，伸直了她又白双胖的手臂装作一本正经的样子将钱递给我，手还紧紧抓着钱。我忍不住笑了起来，见钱眼开，从她手里拔出了钱。这时徐安淇偷偷拿走了橡皮，我指了指她，施蕴嘉立刻回过神来，抢橡皮去了。

这就是我卖掉的第一份货物，这一次的成功，也给了我下一次也能卖出去的信心。

使用块来做闭包



在Ruby中可以使用块(Blocks)来实现闭包(Closure)的功能，闭包听过很多回了，但都没仔细去看过，看维基中有关闭包的定义 )：

在计算机科学中，闭包（Closure）是词法闭包（Lexical Closure）的简称，是引用了自由变量的函数。这个被引用的自由变量将和这个函数一同存在，即使已经离开了创造它的环境也不例外。所以，有另一种说法认为闭包是由函数和与其相关的引用环境组合而成的实体。

太拗口了。不过简单来看，闭包不仅仅是函数，还包括了函数的上下文，而使用闭包可以使得你在函数的外部也引用函数体内的变量等上下文环境。

看一个例子：

def nTimes(aThing)
return proc { |n| aThing * n }
end

p1 = nTimes(23)
p1.call(3)
»
69
p1.call(4)
»
92
p2 = nTimes(“Hello “)
p2.call(3)
»
“Hello Hello Hello “
</pre> 方法nTimes返回了一个Proc对象，其中引用了aThing这个方法中的变量，而在外部调用p1时仍可以使用aThings，这就是闭包。 在[闭包的概念、形式与应用](http://www.ibm.com/developerworks/cn/linux/l-cn-closure/)中有一段Ruby中闭包的描述： _所谓 Block 是指紧跟在函数调用之后用大括号或 do/end 括起来的代码，Block 的开始部分（左大括号或 do）必须和函数调用在同一行。Block 也可以接受参数，参数列表必须用两个竖杠括起来放在最前面。Block 会被作为它前面的函数调用的参数，而在这个函数中可以使用关键字 yield 来调用该 Block。_ _Block 实际上就是匿名函数，它可以被调用，可以捕获上下文。由于语法上要求 Block 必须出现在函数调用的后面，所以 Block 不能直接作为函数的的返回值。要想从一个函数中返回 Block，必须使用 proc 或 lambda 函数把 Block 转化为对象才行。_ 阮一峰也有篇文章[学习Javascript闭包（Closure）](http://www.ruanyifeng.com/blog/2009/08/learning_javascript_closures.html)提到了闭包的作用：“一个是前面提到的可以读取函数内部的变量，另一个就是让这些变量的值始终保持在内存中。” 读取函数内部变量之前的例子就可以看到，再写段测试代码验证变量的值保存在内存中。 <pre>

公元纪年

讲历史绕不开一个公元纪年，在和小乐画时间阶梯时大量的历史事件都是发生在公元前。

那么公元又是什么呢？

给小乐念“天国、权力、荣耀，全是你的”，就说到了：

耶稣诞生之前都称为公元纪年，他诞生之后称为公元。你一定认为他诞生的时间就是0年了。

不过事实上，公元纪年的起点时公元1年，而没有公元0年。

 

而更有意思的是，大约在公元500多年后，人们才开始以耶稣诞生的时间来纪年，可他们在开始纪年的时候，把时间给弄错了。后来人们才发现耶稣实际上生于公元纪年的4年之前，也就是说，在公元前4年，但是，等发现这个错误的时候，已经来不及改正了。

 

“这真是个美丽的错误，错了要改就很困难了。比如说有人是600年出生的，结果要和他说你是596年出生的，有的知道要改，有的不知道，那就都要错乱了。”

“那就是说，我实际上应该是1999年出生了。”

 

维基百科中有关公元的说明：公元525年，基督教神学家狄奧尼修斯·伊希格斯建议將耶穌生年定为纪元元年，以取代当时罗马教廷所採用的「戴克里先历」。伊希格斯並推算耶穌是生于罗马建国后754年，但后來的历史学家发现他的计算有误；現在一般以耶穌誕诞生在公元前7年～前4年左右的说法最可靠。

看来这还不是个准数啊。

 

中国是在辛亥革命后次年1912年，中华民国政府决定采用西历历法（阳历）作为国历，但纪元部分沿用中国传统，采用中华民国纪年。

1949年中国人民共和国成立后放弃使用民国纪年，改采用世界通用的公元纪年制度。

君士坦丁和礼拜天的由来

今天小乐挑了个180的数字，就从180页开始讲。文章的题目很怪，希利尔说这个故事的题目要到最后才说。

“为什么？”

因为除非你听完这个故事，否则根本没法明白题目的意思，所以提前看题目也没用。

“那我更想先知道这个题目了。”

“OK.那我先告诉你题目是I_HS_V__。”

“啊，那是什么意思啊？”

“In Hoc Signo Vinces(以此为记，必将得胜)”

“不懂。”

那让我们来开始今天的故事吧。

大约在公元300年，罗马有个皇帝叫君士坦丁)，他不是基督徒。

“君士坦丁堡？”

嗯，君士坦丁信奉的是罗马的那些神灵，不过可能他对这些神也没多少信仰。

有一次，君士坦丁带兵打战，做梦看见天空中有一个火焰般的十字架，十字架下方有一行拉丁文字“In Hoc Signo Vinces”意思是以此为记，必将得胜。君士坦丁认为如果他带十字架参加战斗，就一定能征服敌人。他就想试试也没坏处，就让士兵盾牌都做个十字架标记上阵杀敌，结果还真打赢了。罗马元老院为此还特地建了一座凯旋门叫君士坦丁凯旋门，就在罗马的广场上，有三个圆拱组成。

之后君士坦丁就宣布基督教在罗马帝国合法化，据说他死前正式受洗成为一名基督徒。

君士坦丁对罗马城不是很喜欢，更愿意住在罗马帝国东部的一个城市拜占庭，就搬到那里去住，并作为都城。拜占庭又叫新罗马，后来干脆叫君士坦丁的城，希腊语中表示城市的词叫polis（音波利斯，表示城邦），例如Annapolis美国的安纳波利斯，Indianapolis美国的印第安纳波利斯，所以君士坦丁的城就是Constantinepolis，后来简化为Constantinople，也就是君士坦丁堡。

在君士坦丁时代以前，罗马帝国每周是没有假日的，星期天没什么不同，该咋的还咋的。君士坦丁认为基督徒每周得有一天来礼拜上帝，这天是个神圣的日子，用英文表示就是holyday，简化holiday，也就是假日。

“就是礼拜日？”

所以君士坦丁把星期天定为基督教的休息日，这一天他们不用工作，专门礼拜上帝，就如星期六是犹太人的圣日，星期五是穆斯林的圣日一样。

给小乐讲“希利尔讲世界史”之英国大宪章

给小乐说十字军东征的时候是先说“没人喜欢的约翰”，结果跳到了约翰的兄弟狮心王理查德，现在继续回到没人喜欢的约翰。约翰是历史上又一个讨人厌的恶棍，我们都不喜欢他，但愿意听人讲他的故事，当听到他得到应有的下场时，会高兴得鼓掌欢呼。

约翰杀死了他的侄子亚瑟，怕他会代替自己做国王；想让自己的朋友做英国的主教，和罗马教皇发生了一场争执，结果教皇命令英国所有的教堂都关闭。英国人收到了震骇，人人都责怪约翰。教皇也威胁要另外选个人来代替约翰做英国国王，把约翰吓坏了，赶紧认错。

希利尔说约翰是个猪脑袋，总是做这样那样的错事。他觉得地球是围着国王转的，其他人来到世界上就是来让国王有佣人使唤的。约翰不停地从富人们那里要钱，要多少就得给多少，不给就用刑，甚至处死。

约翰的恶行变本加厉，最后贵族们再也无法忍受了。于是他们把约翰关起来，并挟持到泰晤士河畔一个叫伦尼米德的小岛上，迫使约翰同意签订他们用拉丁语写成的一些文件。这是在1215年，对约翰来说是个倒霉日子，而对英国人民来说是个黄道吉日。贵族们逼迫约翰签订的那份协议背称作《大宪章》。

 

约翰连自己的名字都不会写，所以他不能象现在的人那样签字。不过，他戴着一个印章戒指，这种戒指就是给那些不会签名的人用的。要签名的时候，只要把一些烧化的蜡油滴到协议上需要签名的地方，然后把印章往蜡油上一按就行了。

在大宪章里，约翰同意赋予贵族们一些权利，而我们现在来看，这些权利是每个人都应该无条件拥有的。比如说：人们当然有权保管自己的财产，也有权让这些财产不被他人非法夺走；除非人们确实做错了事情并受到了公平的审讯，否则不应被关进监狱，也不应被国王或任何人处罚。这就是《大宪章》里规定的两项权利，此外还有很多类似的条约。

“这些权利在现在大家可能觉得是基本的，但以前可很了不起。你觉得财产是不是应该是自己的，要受到保护？”

“是的。”

“可是国王想夺走就夺走。”

“或者政府。”

“是的，比如你有一块土地，也许政府看上了，就会想要你的地，当然他可能会出点钱，比如100块，但这地可远远不只这个钱。还有人身安全，必须受到公平审讯后才能被处罚，要知道以前可都是国王说了算，我们国家也一样是皇帝说了算。”

 

1215年的大宪章以法律限制了英国王室的绝对权利。不少日后編成的政府宪法，包括美国宪法，都是起源自大宪章。

块和迭代器



块（Block）就是一块用{}或do/end括起来的一段代码，可以使用块来实现回调(callback)、迭代器(iterator)。

采用yield在方法中去调用块。

def callback
yield
yield
end
callback { puts “In the block” }
</pre> 调用yield的时候可以传递参数，在block里，使用“||”列出参数。 迭代器是针对容器（比如Array）中的连续的元素重复调用一个块的代码。 块中最后一个表达式的值作为返回值会返回给调用方法，作为yield的值。比如find： <pre>
class Array
def find
for i in 0…size
value = self[i]
return value if yield(value)
end
return nil
end
end

[1, 3, 5, 7, 9].find {|v| vv > 30 } »7
</pre> 再看一个例子，实现了数组元素的累加和阶乘，这里each是个迭代器，inject也是个迭代器。 <pre>
class Array
def inject(n)
each { |value| n = yield(n, value) }
n
end
def sum
inject(0) { |n, value| n + value }
end
def product
inject(1) { |n, value| n value }
end
end
[ 1, 2, 3, 4, 5 ].sum »15
[ 1, 2, 3, 4, 5 ].product »120
</pre> 块可以用在一些事务控制中，比如打开文件，做某些事情，然后关闭，简单的做法可以使用块，打开文件，调用块，将文件句柄传递给块，当块返回时，关闭文件。这样就把关闭文件的责任从文件的使用者交给了文件对象本身。如： <pre>
class File
def File.openAndProcess(args)
f = File.open(args)
yield f
f.close()
end
end

File.openAndProcess(“testfile”, “r”) do |aFile|
print while aFile.gets
end

童心

每年高考作文结束后，都照例会有一堆的讨论，范文，不管是高分的还是搞笑的，以及对各高考题目的评头论足。

钱江晚报9日的“众议高考”有两篇评论：
一篇是“失去孩提王国，教育也将沉沦”，举到一个例子，教学是标准化的，题目要求和“照耀”组词，标准答案是“阳光”，答了“太阳”就是错。作文在一定程度上也是标准化的，而标准化的第一条原则就是“主题思想正确”，或“中心观点正确”，而不是感情的真实、观点的真实。
想到小乐的那个有趣的夏天的故事，最后的故事结尾是西瓜被老师给瓜分了，看来是属于思想不正确了。

一篇评论是“教育与学生一样渴望解放童心”，其中说到“高考作文的假大空是果，又是因，教师与学生同时背这恶性循环裹挟着”。小学生作文竟然无话可说，有人归咎于“教师指导作文不力”，其实恰恰相反，是“太用力”了，早早让小学生作文内容戴上了面具，结构戴上镣铐，让他们背负内容与形式的庞大教条，乃至于人生说谎作文始，人生老成作文始，怎么可能让他们活泼的童心、纯真的情感保鲜。
小乐的作文在写着，写着的时候，我常说她有点虎头蛇尾，开篇往往比较宏伟，但到最后不知是急于要收尾呢，还是要套入”啊XXXX，多么XXX，我好XXX”的抒发感叹系列中，这实际也是结构的标准化。

其实学校与老师们也深刻地懂的这个道理，但他们也受制于这个教育体系，考核学校、考核老师的KPI指标是什么呢，还是考试的分数，所以最终走向了标准化，而忽视了童心的宝贵。

给小乐讲了个美国幼儿园起诉案的故事，是在朋友圈中看到的，说的是美国幼儿园的一位母亲因为女儿认识礼品盒上的一个字母”O”，而把幼儿园告上了法庭。因为幼儿园教孩子认识”O”的同时，剥夺了孩子的想象力，而孩子本来会把”O”说成苹果、太阳、足球等等圆形的东西，现在孩子的这种能力失去了。最后这位母亲赢得了这场官司，她在辩护中又讲了一个故事：
“我曾到东方某个国家旅行，在一家公园里见过两只天鹅，一只被剪去了左边的翅膀，一只完好无损。剪去翅膀的被放养在较大的一片水塘里，完好的一只被放养在一片较小的水塘里。当时我非常不解，就请教那里的管理人员。他们说，这样能防止它们逃跑。我问为什么?他们解释，剪去一边翅膀的无法保持身体平衡，飞起后就会掉下来；在小水塘里，虽然没被剪去翅膀，但起飞时会因没有必要的滑翔路程，而老实地呆在水里。当时我非常震惊,震惊于东方人的聪明。可是我也感到非常悲哀，为两只天鹅感到悲哀。今天，我为我女儿的事来打这场官司，是因为我感到伊迪丝变成了劳拉三世幼儿园的一只天鹅。他们剪掉了伊迪丝的一只翅膀，一只幻想的翅膀，人们早早地就把她投进了那片小水塘，那片只有ABC的小水塘。”

对这个故事的真实性，我有一些怀疑，不过和小乐讲这个故事，还是想和小家伙说童心的可贵，不必追求所谓的对错，同样真实也远胜于华美。

钱报的评论说：这样看来，今年浙江的高考作文题出得真好：“世界正在失去伟大的孩提王国。一旦失去这一王国，那是真正的沉沦。”

小乐听完故事后，诗兴大发，写了首：

童心在哪里

童心，在孩子的言语里。

一声亲切的问候，

一声快乐的欢笑，

都包含着属于孩子的童心。

童心，在孩子的行为里。

一个善意的恶作剧，

一张有趣的贺卡，

都隐藏着强大童心。

童心，还在许多地方，

其实大人也能拥有童心，

和孩子一样的童心，

只是你要把童心找回来，找回来…

小乐

时间阶梯

历史的阶梯是比这幅图的底部还要更遥远、更遥远的过去开始，一直这样向上、向上、向上，直到我们现在的位置——每一个台阶都是一百年，每一层楼梯是一千年。这个楼梯可以一直不断地向前升，直到高空。让我们站在现在的位置，回顾下面的楼层，倾听那些发生在过去漫长岁月中的故事。
这个就是时间楼梯，希利尔在《希利尔讲世界史》中专门介绍了这样一个阶梯形状的图标能使孩子对于世界历史的时间范围和发展阶段有一个形象的认识。

看到这个图的时候，我就觉得这玩意真不错，画一个在经常看见的地方，能直观地感受到重大事件在历史长河中的位置，并且也不会觉得那么孤立。于是趁着假期，就鼓动小乐一起动手，参照书中的附图，画了一个“时间阶梯”图，从石器时代一直到公元2000年，一层是1000年，一个台阶100年，画的时候感觉时间好短暂，我就这么轻轻一横100年就那么过去了，呵呵。在每个100年的事件标注中除了参照原图中的重大事件，抹去了一些我们不熟悉的内容，同时最重要的是我们加入了中国历史朝代。

不知道大家在学习历史的过程中有没有这样的感觉，就是中国历史和世界历史是脱节的，以前学历史的时候也总是背843年查理曼帝国一分为三，1789年7月14日法国大革命，但从来没搞清当时的中国是在干嘛？所在在和小乐画时间阶梯的时候，我们把同一朝代在阶梯上用相同颜色的线段加粗，同时标注朝代的起止时间，这样我们就看到了希腊举行第一次奥林匹克运动会的时候是在西周，十字军东征是在南宋的时候，有趣吧！

希利尔建议有必要让孩子们把“时间楼梯”牢牢记住，就像“九九乘法表”一样，经常记忆，知道能够详细说出与每个日期相关联的时间。这样做的目的是让孩子们能够对从古至今的世界历史做出一个大致的概括，并可以说出重要的历史日期和历史事件，不会由犹豫，不犯错误。听起来有点难，但值得尝试一下，我再想想办法看，大家有什么好方法也能提提建议。

坚持100天学习Ruby

第一周的总结



6月3日开始坚持100天学习Ruby，到现在刚好一周，总结一下。

看了一篇20分钟入门的教程，另外Programming Ruby - The Pragmatic Programmer’s Guide看了前言和前三章。 每天基本上还是坚持下来，主要的时间还是中午休息时间，不过由于有两天会议拖得太长，需要另外再找时间。看书一个番茄时间，但整理内容就要超出了。
前面基本是些基本知识，也比较简单，后面的任务会更艰巨；另外考虑光看也不够，每周想还得做一两个实例。

把看到几点再聚集一下： Ruby是一门真正的面向对象的语言，任何操作的东西都是对象，包括操作结果本身，但注意变量是对象引用。
对象的方法调用传递的是消息。 单引号所括的内容不会做转义，直接就是字符串的内容；而双引号会做值替换。
变量的命名第一个字符，@是实例变量，@@是类变量，$是全局变量。 类从没有被关闭，你可以始终给已存在的类继续添加方法。

还不清楚的点，一个是mixin，一个是singleton例子的非线程安全问题。
