mac 上邮件客户端的选择

mac上提供的mail.app还是很赞的，直接输入邮箱即可连接公司的Exchage Server，基本上用的很习惯。

然后碰上了第一个问题：我习惯把收件箱中的邮件看完后即转移到archive目录，我希望能有一个快捷键，类似gmail中的y。找了和试了一些方法，在twitter上向xiaolai求助了一下，得到了解决方法。Lion直接提供快捷键设置，可以设置全局的，也可以针对某一个app进行设置。

不过这个方法不是很完美，就是在mail.app的菜单中会有两个相同的“archive”菜单项，一个是move，一个是copy，经常会引起混乱。所以我始终觉得不是很爽。

继续使用的过程中，无法接受的问题出现了。居然我近期发送的带附件的邮件同事们在outlook中查看时，附件或图片的后续文本都被当成了ATT类的附件。这个问题很头疼，没找到好的解决方案，所以我只好尝试其他邮件客户端了。

以前用过的两个选择：Thunderbird，Outlook都试了一下。

Thunderbird在windows上当初曾用了一年多，也试过一些插件，还是不错的。但很难接受的一点是和office之间的表格、图片的拷贝不支持。（这一点也比不上mail.app）另外总觉得界面风格也有点丑，不习惯。

Outlook之前打算做过配置的，结果设置账户为Exchange的时候，但悲催地得到了提示outlook 2011不被我们的Exchagen Server支持，需要exchange 2007 sp1。给IT反馈暂时也没用啊，当时就直接放弃了。现在碰到这个问题后，只好重新回过来再看Outlook，设置账户为IMAP方式，这样就能搞定了，快捷键的问题还是象上图一样搞定。

有缺憾的问题是非Exchange账户连接方式，需要自己去设置和服务端同步的IMAP文件夹，日历、规则和通讯录就无法直接和服务器同步了。只能要这样了，一是等公司的IT升级；二是等mail.app和outlook之间的联通性了。

freemind/filezilla/dropbox在mac下使用的几个问题

之前养成好习惯，尽量使用一些跨平台的软件，比如Evernote, dropbox, Filezilla, Freemind，这样在切换到mac上的时候能加快过渡。

不过在实际使用的时候，在这几个软件上都碰到了一些问题：
1. Freemind无法输入中文
解决方法：先切换到英文输入发，再启动Freemind，就可以输入中文了；
2. Filezilla连接上ftp server后，不能显示目录
解决方法：今天服务器上传了一些去苏州玩的照片，结果上去一看服务器上啥文件也看不到。原来是编码的问题，在站点管理器中修改站点的编码为gb2312就可以了；
3. dropbox无法实时更新
解决方法：这个应该和mac无关，只是因为一下子有了两台机器实时更新的需求，才发现它的存在和严重性。月光博客上有篇详细的文章，还是要感谢伟大的GFW。没用privoxy，我暂时还没去管它。

在MAC翻墙遇到的问题

原先有ssh的账号，在mac上翻墙倒也不麻烦，下了个secret socks，略微配置一下连上usassh。

直接勾选“Update network settings”，提示输入root口令后，就会直接修改网络中的设置以sock方式访问。这时打开safari就可以翻墙了。
不过这种方式的缺陷还是所有的站点访问都会走ssh，国内很多站点的访问速度会有影响。还是想能改成智能点的，只有自己需要的站点访问就好了。
采用PAC的方案，我把原先chrome中的proxy switchy设置的列表导出成一个pac文件，然后在网络中设置代理方式为“自动代理配置”，同时提供配置文件。
从google的结果来看都说很简单，这样就ok了，可是俺死活没搞通。瞎折腾了半天，要么都访问不了，要么就是走的ssh通道。
奇了个怪了！
后来忽然想是否我的pac文件不对，于是自己写了个最简单的pac文件：
function FindProxyForURL(url, host)
{
        return “SOCKS 127.0.0.1:1080”;
}
换了个思路，也就找到了问题的根源。才发现是犯了两毛病：
1. 我最开始在代理配置文件中的URL提供的是“file://”形式的地址，采用了“选取文件”的按钮，貌似lion不支持“file://”形式。嗯，这个就启用网络共享，提供http的访问形式。
2. 从chrome导出的pac文件采用的sock地址形式都是”SOCKS5 127.0.0.1:1080”，而不是SOCKS。规格的说法应该是提供SOCKS。一字之差害死人啊。
终于在晚上睡觉前搞定了，可以安心睡觉了。

西安旅游的应用篇

在小乐暑假快要结束的时候，全家人一起去了一次西安，也就有了小乐长安行。之前在微博上曾提到了西安旅行一个很大的感受就是移动互联生活的便捷，给这次旅游带来了很大的方便性，旅行质量也提高了好多。甚至LP大人在此期间都不那么烦我一天到晚拿个手机，哈哈。

整理一下做个推荐：

  1. 大众点评 出门第一担心的就是吃饭的事情，吃什么呢？贵不贵？好不好？远不远？有没有西安特色？现在全部都不用担心。我在西安的正餐基本上都是大众点评推荐解决的，每次快要逛完景点了，就开始搜索附近500米、西安风味的餐馆，然后只要根据点评进行挑选就可以了。选择餐馆之后，再直接获取路线，就可以在地图上得到信息，左转，直行，感觉自己就是个本地人啊。在小乐面前别提有多神气了。
  2. 百度地图 其实iphone上本身的地图已经很好了，尤其是和其他应用的结合。不过出行之前我还是多选了个百度地图做后备。一来是担心Google地图在我国的特殊性，二来就是百度地图提供了离线地图的方式，我在出行前把西安和咸阳的地图都下载了。这样既省了流量，也不会因网络信号差而显示不了地图。
  3. 旅游攻略 出门前挑了挺多有关旅游的APP的，什么全国旅游景点、无线西安、陕西旅游，发现大多不靠谱，没啥实质内容。直到下了旅游攻略中，我就看了西安的，从概述、文化、交通、小吃、沟通无、旅游路线、景点介绍一应俱全，靠谱！充分体现了“内容为王”啊！另外制作也比较精美，推荐！不过我也给它提了个意见：特讨厌每次进旅游攻略，它就要提示来一下是否要联网更新旅游攻略列表？然后下面两按钮就是“客观，我不要！”，“我要！”搞得跟什么官人我要似的，特烦人。（BTW，貌似我们的APPSTORE也是类似问题，总想不断地提示别人来更新，可要是内容不行，有人来吗？）
  4. 随手记 金蝶出品，还可以同步到服务端，原先是想用来日常做记账的，不过记了没几次，就懒了。没想到这次出去旅游倒是发挥了作用，可以随手记账，所有旅行费用支出一览无遗，不用担心哪里钱数目不对了。一到家，整个这次的旅行消费就全出来了。同时还提供简单统计图表，如图：貌似以后还可以进一步先制定各块的预算，严格控制，哈哈。
  5. 其他的APP虽然下了不少，但真正起到作用的倒也还不多。比如锦江之星只能提供查询，海南航空虽然可以办理业务，但却没有把我在网站上办理的信息给同步过来，我实在是没想通。航班、公交、铁路基本都没用上，百度地图中也提供了公交线路的查询，差不多也够用了。
  6. 在网络上搞定的事情，有些虽然不是用APP，但有浏览器就能搞定很多事情了。比如航班的自助登机，可以提前选号我们一家三口的座位，就不用担心分开了。
    对了，还有必须说明的应用就是新浪微博了（我用好多个客户端，下次找个时间比较一下），旅游中多了个小乐长安行的随时播报，这也是以往从来不曾有过的。一般我都是在坐车的时候，记账、发微博，哈哈。

遗憾嘛，之前也说了，网络是基础，还是不够给力！

陪小乐做聪明题

上周末的时候小乐的聪明题里有这样一道题目：

数学-学数=36，然后要填：数=( )，学=( )

从大人的思维来解这道题目：

列方程：10x+y-(10y+x)=36

化简后得：9(x-y)=36，也就是x-y=4

所以答案应该是有多组解：(5,1),(6,2),(7,3)…

不过怎么和小乐说呢？

先还是从凑数字开始吧，我们一起凑了：

数=5，6，7

学=1，2，3

“有什么规律呢？”

答案不言而喻，小乐说：上面是5，6，7，依次往下数，下面是1，2，3，依次往下数。

所以我们就又很容易得到了84，95。

“还有什么规律吗？”

4-1，5-2，6-3，两个数相减都为4。

不过这时候，小乐又凑出的一个数字就让我吃惊了一下，因为她给出了40-04，这一下子就突破了我的惯性思维，总以为二位数是是没有0的。这使得题目也好解了很多，变成了两位数减一位数。也使得我重新再出一道题目的时候，可以很容易从0开始算起。

比如：数学-学数=27

小乐又从0开始推，很简单就得到了30-03=27，然后之后就是十位数，个位数往上加1了。使得我讲题目一下子轻松了好多，这个0实在是太秒了，击掌祝贺一下！

那么是不是题目就到此结束了呢？

其实还不然，这时候，我让小乐自己来出一道题目看看，出一道题目给她老妈做。这时候可就又有点难了！下面的差也不是随便能凑的，呵呵。

于是，我们就一起继续找下面差的规律，36，27，18…

小乐告诉我的第一个规律，又让我吃惊了一下，她说两个数字相减是3，5，7，都是奇数。哈哈，果然是，这又是我没想到的。

“我们再看看两个数相加会怎样？”

3+6，1+7，1+8，都是9，所以如果再出一个题目的话可以是45，54。

这里的答案其实在上面的式子中9(x-y)=36，要求减数必须是9的倍数。不过这个等以后再和小乐再说了。

数学还是很有趣的，陪小乐的时间太少了点，不然一定能一起找更多有趣的规律来。
