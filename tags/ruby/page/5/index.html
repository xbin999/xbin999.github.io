块和迭代器



块（Block）就是一块用{}或do/end括起来的一段代码，可以使用块来实现回调(callback)、迭代器(iterator)。

采用yield在方法中去调用块。

def callback
yield
yield
end
callback { puts “In the block” }
</pre> 调用yield的时候可以传递参数，在block里，使用“||”列出参数。 迭代器是针对容器（比如Array）中的连续的元素重复调用一个块的代码。 块中最后一个表达式的值作为返回值会返回给调用方法，作为yield的值。比如find： <pre>
class Array
def find
for i in 0…size
value = self[i]
return value if yield(value)
end
return nil
end
end

[1, 3, 5, 7, 9].find {|v| vv > 30 } »7
</pre> 再看一个例子，实现了数组元素的累加和阶乘，这里each是个迭代器，inject也是个迭代器。 <pre>
class Array
def inject(n)
each { |value| n = yield(n, value) }
n
end
def sum
inject(0) { |n, value| n + value }
end
def product
inject(1) { |n, value| n value }
end
end
[ 1, 2, 3, 4, 5 ].sum »15
[ 1, 2, 3, 4, 5 ].product »120
</pre> 块可以用在一些事务控制中，比如打开文件，做某些事情，然后关闭，简单的做法可以使用块，打开文件，调用块，将文件句柄传递给块，当块返回时，关闭文件。这样就把关闭文件的责任从文件的使用者交给了文件对象本身。如： <pre>
class File
def File.openAndProcess(args)
f = File.open(args)
yield f
f.close()
end
end

File.openAndProcess(“testfile”, “r”) do |aFile|
print while aFile.gets
end

Ruby中单引号和双引号的区别



Ruby中使用引号括起字符表示字符串，单引号和双引号有所不同，单引号所括的内容不会做转义，直接就是字符串的内容；而在双引号中，Ruby会做更多事情。首先Ruby会查找转义符‘\’，并做相应的替换，典型的如换行’\n’；其次是表达式求值，对于字符串中的#{expression}会被表达式的值所替换。

1.9.3p125 :060 > puts “Hello\nworld”
Hello
world
=> nil
1.9.3p125 :061 > puts ‘Hello\nworld’
Hello\nworld
=> nil
1.9.3p125 :062 > name = “Ted”
=> “Ted”
1.9.3p125 :063 > puts “Hello,#{name}”
Hello,Ted
=> nil
1.9.3p125 :064 > puts ‘Hello,#{name}’
Hello,#{name}
=> nil

Ruby和其他面向对象语言的一点差异



Ruby是一门真正的面向对象的语言，任何操作的东西都是对象，包括操作结果本身。

song1 = Song.new(“Ruby Tuesday”)
song2 = Song.new(“Enveloped in Python”)
# and so on
</pre> 在Ruby中定义一个类(class)，通过constructor new来创建对象(object)，这些实例(instance)从同一个类衍生，但有不同的唯一标识。你可以定义不同的实例变量，持有对象的状态，比如每首歌。 每个类可以定义实例方法，以存取对象的实例变量。通过给对象发送一个消息来调用方法，消息包括方法名称，所需要的参数。当对象接收到一个消息，它会查找类本身是否能响应方法，如果能找到，执行方法。 看下面的例子： <pre>
“gin joint”.length » 9
“Rick”.index(“c”) » 2
-1942.abs » 1942
sam.play(aSong) » “duh dum, da dum de dum …”

点前面的称为接收器(receiver)，之后的是被调用的方法。

这里有一点Ruby和大多数其他语言的不同。比如求绝对值，Java会调用一个独立的函数并传递参数：

number = Math.abs(number) // Java code

但在Ruby中，求绝对值是内置在数字本身的，内部关注细节和完成处理，你只需要简单地发送一个abs消息给数值对象即可。

# 免费的计算器

我们可以把IRB当作一个计算器使用
<pre>`
irb(main):003:0&gt; 3+2
=&gt; 5
irb(main):004:0&gt; 3_2
=&gt; 6
irb(main):005:0&gt; 3_*2
=&gt; 9
irb(main):006:0&gt; Math.sqrt(9)
=&gt; 3.0
`</pre>
在Ruby里**表示乘方，而Math.sqrt指的是开平方根。那么Math是什么呢？

Math是内置的算术模块（Module）。模块在Ruby中两个角色，这里演示了其中的一个，就是把相似的方法组织在同一个名字下。象Math中也包括sin()和tan()。

点“.”用来标识一个消息的接收者。什么是消息？在这个例子中就是sqrt(9)，表示调用方法sqrt。

计算的结果是3.0，而不是3，因为大多数情况下开平方根不是整型，所以该方法的返回值是浮点型数值。

如果我们希望把计算结果保存下来，那么就赋值给一个变量。
<pre>`
irb(main):007:0&gt; a = 3 ** 2
=&gt; 9
irb(main):008:0&gt; b = 4 ** 2
=&gt; 16
irb(main):009:0&gt; Math.sqrt(a+b) =&gt; 5.0
`</pre>

# 更强大的MegaGreeter

上面的greeter还不够有趣，一次只能处理一个人。我们可以编写一个更强大的MegaGreeter，可以和world打招呼，也可以向一个人或者一堆人打招呼。

接下来退出IRB，编写一个独立的文件。
<pre>`
#!/usr/bin/env ruby

class MegaGreeter
  attr_accessor :names

  # Create the object
  def initialize(names = "World")
    @names = names
  end

  # Say hi to everybody
  def say_hi
    if @names.nil?
      puts "..."
    elsif @names.respond_to?("each")
      # @names is a list of some kind, iterate!
      @names.each do |name|
        puts "Hello #{name}!"
      end
    else
      puts "Hello #{@names}!"
    end
  end

  # Say bye to everybody
  def say_bye
    if @names.nil?
      puts "..."
    elsif @names.respond_to?("join")
      # Join the list elements with commas
      puts "Goodbye #{@names.join(", ")}.  Come back soon!"
    else
      puts "Goodbye #{@names}.  Come back soon!"
    end
  end

end

if __FILE__ == $0
  mg = MegaGreeter.new
  mg.say_hi
  mg.say_bye

  # Change name to be "Zeke"
  mg.names = "Zeke"
  mg.say_hi
  mg.say_bye

  # Change the name to an array of names
  mg.names = ["Albert", "Brenda", "Charles",
    "Dave", "Engelbert"]
  mg.say_hi
  mg.say_bye

  # Change to nil
  mg.names = nil
  mg.say_hi
  mg.say_bye
end
`</pre>

分析一下新的程序，Ruby中#开头表示注释，第一行是个特殊例子，在类Unix系统中用于告诉shell如何运行这个文件。

say_hi中先判断@names实例变量，如果是空则打印三个点。如果能响应each方法，则进行迭代，比起for循环对元素一个个操作来说更优雅，不用定义循环变量，只需要告诉它怎恶魔处理每一个元素就行了。

块（block）的强大在于处理比list更复杂的东西，除了处理一些方法中的简单细节外，你还可以处理setup, teardown和错误。在say_bye中没有使用each，而是检查@names是否响应join方法，如果是的就调用它。这个方法不需要关心变量的真实类型，而只需要看它是否支持join方法即可。

if FILE == $0中的FILE是一个神奇的变量，其值是当前文件名称，这个判断用于表示“如果运行调用的是本文件”。这可以让一个文件作为库(library)使用，而不会被执行；但如果被当作可执行文件调用市，就执行相应的代码。

编写markdown文档直接发布到wordpress上



我现在用nvALT来书写markdown文档，有时想把写好的文章发布到自己的博客上，比较土的方式往往是先预览，然后打开博客编辑，复制粘帖。

想想是不是可以简化一下，能直接把写好的markdown文档发布到wordpress博客上。试了一下，分三步走：

1. 可以把nvALT的存储模式设置成普通文本文件（Plain Text Files ），这样写好的内容就会以单个文件形式保存在设置的目录中。
2. 然后使用flavor 讲文件转换成html文件，需要安装ruby 和json。
3. 发布到wordpress上，这个原来想偷点懒，直接基于wordpress提供的发布邮件账号，通过邮件发出即可，但后来发现直接基于命令行上的mail命令，不支持html文本。只好另外想办法，找个API来直接调用接口发布。wordpress支持xmlrpc，有个简单的例子Simple WordPress Posting from Ruby via XMLRPC，直接搬来使用即可。
4. 最后做的事情就是把flavor和上面的例子串接一下，代码放在github上。
这篇文档就是以这个方式完成发布的，测试的过程中碰到几个问题： 运行flavor时报错

~/.rvm/gems/ruby-1.9.3-p125/gems/json-1.7.5/lib/json/ext/parser.bundle: [BUG] Segmentation fault
ruby 1.8.7 (2012-02-08 patchlevel 358) [universal-darwin12.0]
</pre> 这个原因是flavor.rb指定了/usr/bin/ruby，和我的rvm环境不一致，修改ruby运行环境即可。 * 命令行下mail 运行时报错 <pre>sendmail: fatal: chdir /Library/Server/Mail/Data/spool: No such file or directory

这里需要设置mac下的mail环境，有篇详细的说明。
* 中文字符的问题

xmlrpc/create.rb:46:in `join’: incompatible character encodings: UTF–8 and ASCII–8BIT (Encoding::CompatibilityError)

这里需要讲flavor调用github解析返回的结果做强制编码html.force_encoding(“UTF–8”).encode(“UTF–8”)。
