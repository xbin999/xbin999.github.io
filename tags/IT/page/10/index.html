升级到Firefox 2.0



Firefox 2.0推出好久了，一直都没时间去更新，看到也有不稳定的事，咱就不去赶这个鲜了。

现在火候估摸着差不多了，我一直用的是Portable Firefox，今天去了看，还是2.0 RC3。也不管了，直接去下了个Firefox 2.0，安装了之后把整个目录复制到我的Portable的firefox目录下，然后再把安装的给卸了。

重新启动Portable Firefox，原来的extensions, plugins, profile都还在，只是有几个扩展不兼容了，基本OK。

直接使用了Firefox自己的tab浏览功能，也不错，双击就新开一个。不过我想把它放在下面的位置，找不到设置:-(

我常用的一些免费工具



上次理过常用工具集合，现在看看又有不少变化了。推荐一些我在用的工具和在线资源吧，关键都是free的，而且很多也是portable的，就不会因盗版而郁闷了。

Gmail & GTDGmail

gmail就不用说了，我现在把公司的邮件都通过它来转发，最大的好处就是过滤垃圾邮件。（公司内部推荐了一下，反应不错）空间也已经超出了2G的限制，我看了一下现在我的gmail空间大小已经是2764MB了，不知最大能到多少？

现在再加上GTDGmail，一个firefox的扩展，这样你可以方便地在gmail中使用label和filter来设置你的action, next action。

Getting Things Done!

mp3tunes

一个在线的音乐存储空间（1G免费），你可以把你的mp3都放上去，然后在线播放，释放你的硬盘空间，呵呵。

Zoundry

上次说了，离线的blog书写工具。现在的文章都是用Zoundry写的了。

Greatnews

离线的rss阅读工具，也是portable的。

Foxit Reader

轻便的pdf阅读工具，比acrobat reader小，占用资源也小，启动更快。

AVG Free Edition

免费的杀毒工具，个人用用应该足够了。

KeePass Password Safe

密码太多了，找个工具管理一下吧。让它随机来给我生成吧。

Tor

洋葱，呵呵。匿名对外连接，摆脱GFW的苦恼，结合Firefox上的扩展xyzproxy，还是挺方便的。

Portable Firefox

这个就不说了，关键还是Portable，不需要每次都去装扩展，习惯能保留。

Portable FileZilla

ftp客户端工具。filezilla另外还有服务器程序。

FreeMind

整理你的思维图，mindmanager的替代。我经常在学东西，或整理系统时用它，可以把思路整理出来，比较清晰，下次备忘。

7-zip

免费的压缩和解压缩工具，替换了winrar和winzip。

Google Calendar & Google Desktop Search

google提供的在线日历管理，我用了也有段时间了，AJAX技术的运用，操作挺方便，速度也蛮快。而且也可以直接google桌面上体现。不过遗憾的是google有时会访问不了，这时calendar就有点不方便了。

桌面搜索是很难离开了，现在找文件或邮件都习惯在搜索中敲文件名或关键字，而很少在explorer中去搜索。所以邮件客户端就不用foxmail了，outlook和gmail的邮件都可以在桌面搜索中直接搜索到。

桌面搜索中也可以添加一些执行项目，对自己做提醒。

foobar2000 & iTunes

foobar2000，一个音频播放工具，支持APE，比较小，资源占用也小。

iTunes，apple的音频播放，和iPod完美结合，可以订阅podcast，就是资源占用大了小。

所以我是两者结合，视机而定。

菠萝网

看看有什么有趣的播客，找到感兴趣的，就在iTunes中订阅。象锵锵三人行，EnglishPod。

豆瓣

老早就推荐过了，还是顺便再写一下。看看人家的书评和影评，再看看自己到底看了些啥，呵呵。

Flashget

挺好用的下载工具，中文免费，也没有网络蚂蚁那么多广告。而且有firefox扩展。

Picasa & Picasa web

图形管理工具，替换了我原先用的acdsee，可以很方便地把照片上传到Picasa web。还可以作些简单的照片拼贴。

先说这么多，下次再整理了。

ILOG Rete Network


培训中的又一个重要关注点，规则引擎的核心算法：RETE Network。
一个入口点；

每条规则一个出口；
RETE Network包含两部分：Discrimination tree和Network of joints。

而在Network of joints有三种节点：

Alpha nodes 保存从Discrimination tree过滤得的节点；

Join nodes 执行条件的join；

Rule nodes 则控制agenda
这是讲义中的例子：

Here is a sample rule:
rule example {
when {
a(fielda1 == 3; ?x: fielda2);
b(fieldb1 == 2; ?y: fieldb2; fieldb3 == ?x);
c(fieldc1 == ?y);
}
then {
execute System.out.println("example");
}
};


Here is the contents of the WM:
a (fielda1 = 3; fielda2 = 10)
b (fieldb1 = 2; fieldb2 = 4; fieldb3 = 10)
b (fieldb1 = 2; fieldb2 = 7; fieldb3 = 10)
c (fieldc1 = 4)
