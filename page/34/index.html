坚持100天学习Ruby

二十分钟学习Ruby

坚持100天学习Ruby的第一天。先从一个教程Ruby in Twenty Minutes 开始，对Ruby有一个整体的印象，20分钟，也在一个番茄时间之内。(不过整理这篇文章还是花了很长时间，下次得再想想方法。)

交互式Ruby

一开始可以在交互式方式下先学习Ruby，打开IRB（Interactive Ruby）

irb(main):001:0&gt;
`</pre>
输入“Hello World”
<pre>`
irb(main):001:0&gt; “Hello World”
=&gt; “Hello World”
`</pre>
我们刚写了著名的“Hello World”程序了吗？不确切，第二行显示的只是IRB返回的最后一个表达式的值。如果需要打印“Hello World”，我们还需要再多做点：
<pre>`
irb(main):002:0&gt; puts “Hello World”
Hello World
=&gt; nil
`</pre>
puts是Ruby中输出的基本命令。而**_=&gt;nil是表达式的返回值，puts总是返回nil_**。

# 免费的计算器

我们可以把IRB当作一个计算器使用
<pre>`
irb(main):003:0&gt; 3+2
=&gt; 5
irb(main):004:0&gt; 3_2
=&gt; 6
irb(main):005:0&gt; 3_*2
=&gt; 9
irb(main):006:0&gt; Math.sqrt(9)
=&gt; 3.0
`</pre>
在Ruby里**表示乘方，而Math.sqrt指的是开平方根。那么Math是什么呢？

Math是内置的算术模块（Module）。模块在Ruby中两个角色，这里演示了其中的一个，就是把相似的方法组织在同一个名字下。象Math中也包括sin()和tan()。

点“.”用来标识一个消息的接收者。什么是消息？在这个例子中就是sqrt(9)，表示调用方法sqrt。

计算的结果是3.0，而不是3，因为大多数情况下开平方根不是整型，所以该方法的返回值是浮点型数值。

如果我们希望把计算结果保存下来，那么就赋值给一个变量。
<pre>`
irb(main):007:0&gt; a = 3 ** 2
=&gt; 9
irb(main):008:0&gt; b = 4 ** 2
=&gt; 16
irb(main):009:0&gt; Math.sqrt(a+b) =&gt; 5.0
`</pre>

# 简单、重复的方法

如果我们想说一堆的“Hello”，而不想把手指敲木，我们需要定义方法（method）。
<pre>`
irb(main):010:0&gt; def h
irb(main):011:1&gt; puts "Hello World!"
irb(main):012:1&gt; end
=&gt; nil
irb(main):013:0&gt; h
Hello World!
=&gt; nil
irb(main):014:0&gt; h()
Hello World!
=&gt; nil
`</pre>
采用def/end定义了一个方法h，然后调用方法。如果方法没有参数，直接调用方法名即可，或者也可以用空括弧()。

如果想对某个人打招呼，而不是对整个世界，只需要重新定义方法h，提供一个name参数。
<pre>`
irb(main):015:0&gt; def h(name)
irb(main):016:1&gt; puts “Hello #{name}!”
irb(main):017:1&gt; end
=&gt; nil
irb(main):018:0&gt; h(“Matz”)
Hello Matz!
=&gt; nil
`</pre>
#{name}是啥玩意？这是Ruby用来在字符串中插入一些东西，大括号中的会被替换成外部的字符串。
<pre>`
irb(main):019:0&gt; def h(name = “World”)
irb(main):020:1&gt; puts “Hello #{name.capitalize}!”
irb(main):021:1&gt; end
=&gt; nil
irb(main):022:0&gt; h “chris”
Hello Chris!
=&gt; nil
irb(main):023:0&gt; h
Hello World!
=&gt; nil
`</pre>
注意两点：一是我们又调用了方法而没有加括号；二是默认参数“World“，也就是说如果你调用时不提供参数，缺省为”World"。

# 创建一个Greeter类

如果你想一个能记住你的名字，能始终欢迎和尊重你，你可以使用一个对象（Object）。让我们来创建一个“Greeter”类：
<pre>`
irb(main):024:0&gt; class Greeter
irb(main):025:1&gt;   def initialize(name = "World")
irb(main):026:2&gt;     @name = name
irb(main):027:2&gt;   end
irb(main):028:1&gt;   def say_hi
irb(main):029:2&gt;     puts "Hi #{@name}!"
irb(main):030:2&gt;   end
irb(main):031:1&gt;   def say_bye
irb(main):032:2&gt;     puts "Bye #{@name}, come back soon."
irb(main):033:2&gt;   end
irb(main):034:1&gt; end
=&gt; nil
`</pre>
采用class定义了一个类Greeter和一组方法，注意_**@name，这是一个实例变量，表示在类的所有方法中都有效**_，就像你在say_hi和say_bye中看到的。

接下来看创建一个对象，并使用它：
<pre>`
irb(main):035:0&gt; g = Greeter.new(“Pat”)
=&gt; #&lt;Greeter:0x16cac @name=“Pat”&gt;
irb(main):036:0&gt; g.say_hi
Hi Pat!
=&gt; nil
irb(main):037:0&gt; g.say_bye
Bye Pat, come back soon.
=&gt; nil
`</pre>
实例变量隐藏在对象中，Ruby采用面向对象机制使数据不直接暴露在外。
<pre>`
irb(main):039:0&gt; Greeter.instance_methods

我们可以看到Greeter对象中有许多方法，而我们只定义了两个方法，其他则来自与父类，可以通过参数false选择不查看父类的方法。我们可以用respond_to?看对象是否能响应指定的方法。

Ruby提供了一种简单的方法用来存取对象的变量。**_使用attr_accessor定义了两个新方法，name用于取得值，name=用户设置值_**。

# 更强大的MegaGreeter

上面的greeter还不够有趣，一次只能处理一个人。我们可以编写一个更强大的MegaGreeter，可以和world打招呼，也可以向一个人或者一堆人打招呼。

接下来退出IRB，编写一个独立的文件。
<pre>`
#!/usr/bin/env ruby

class MegaGreeter
  attr_accessor :names

  # Create the object
  def initialize(names = "World")
    @names = names
  end

  # Say hi to everybody
  def say_hi
    if @names.nil?
      puts "..."
    elsif @names.respond_to?("each")
      # @names is a list of some kind, iterate!
      @names.each do |name|
        puts "Hello #{name}!"
      end
    else
      puts "Hello #{@names}!"
    end
  end

  # Say bye to everybody
  def say_bye
    if @names.nil?
      puts "..."
    elsif @names.respond_to?("join")
      # Join the list elements with commas
      puts "Goodbye #{@names.join(", ")}.  Come back soon!"
    else
      puts "Goodbye #{@names}.  Come back soon!"
    end
  end

end

if __FILE__ == $0
  mg = MegaGreeter.new
  mg.say_hi
  mg.say_bye

  # Change name to be "Zeke"
  mg.names = "Zeke"
  mg.say_hi
  mg.say_bye

  # Change the name to an array of names
  mg.names = ["Albert", "Brenda", "Charles",
    "Dave", "Engelbert"]
  mg.say_hi
  mg.say_bye

  # Change to nil
  mg.names = nil
  mg.say_hi
  mg.say_bye
end
`</pre>
把文件保存为“ri20min.rb”，运行“ruby ri20min.rb”，输出结果如下：
<pre>`
Hello World!
Goodbye World.  Come back soon!
Hello Zeke!
Goodbye Zeke.  Come back soon!
Hello Albert!
Hello Brenda!
Hello Charles!
Hello Dave!
Hello Engelbert!
Goodbye Albert, Brenda, Charles, Dave, Engelbert.  Come
back soon!
...
...

分析一下新的程序，Ruby中#开头表示注释，第一行是个特殊例子，在类Unix系统中用于告诉shell如何运行这个文件。

say_hi中先判断@names实例变量，如果是空则打印三个点。如果能响应each方法，则进行迭代，比起for循环对元素一个个操作来说更优雅，不用定义循环变量，只需要告诉它怎恶魔处理每一个元素就行了。

块（block）的强大在于处理比list更复杂的东西，除了处理一些方法中的简单细节外，你还可以处理setup, teardown和错误。在say_bye中没有使用each，而是检查@names是否响应join方法，如果是的就调用它。这个方法不需要关心变量的真实类型，而只需要看它是否支持join方法即可。

if FILE == $0中的FILE是一个神奇的变量，其值是当前文件名称，这个判断用于表示“如果运行调用的是本文件”。这可以让一个文件作为库(library)使用，而不会被执行；但如果被当作可执行文件调用市，就执行相应的代码。

坚持100天学习Ruby

坚持100天来培养时间管理习惯

@warfalcon 在微信上发起了一个坚持100天的活动，规则是：

  1. 每天坚持完成三个番茄工作法，一次只做一件事。
  2. 每天在早上10点半前完成当天工作中最重要的三件事，这三件事你可以在早上决定这件事是什么？
  3. 梦想番茄：每天用最少1个番茄时间完成你的梦想、重要而不紧急或你一直想做而没有行动的事情，比如学门新的技能、运动、外语、阅读。 具体的事情你自己行动。
    这三项必须要选择一项进行，每天记录，每周总结。

回顾自己在去年的12月份开设了微信平台公众账号“父与女”（fuyunv），目标就是每天陪女儿聊天，讲一个小故事，或者念一首诗，或者说说历史上的今天，每天的时间也就是10～30分钟，然后再记录和梳理，在微信公众平台上进行分享，持续也快有半年了，累计发的文章有150多篇了，看来我的第一个#100天行动#算是完成了，我想接下来应该启动下一个目标。

对番茄工作法并没有太多的了解，@warfalcon 在知乎的回答中有个简短的描述，摘录两句：

  • 番茄工作法原则是25分钟内只做一件事，但如果是几件不重要的小事，可以合并到一个番茄时间内来做。 一件事情超出一个番茄，可以分解成几个番茄。
  • 如果你选择的是梦想番茄，选一件你一直想做的事，从坚持开始每天拿出一个番茄时间，坚持下去。
    我现在比较感兴趣的是梦想番茄，现在的工作基本都是扯的多，做的少，动不动就是说方案怎样怎样，但自己的动手能力越来越弱，而很多时候自己会有些想法，但却很难落入行动。所以我想还是需要学习一项技能，提高自己的动手能力，说白了还是要能快速写点代码，尤其是Web开发方面，目标是能达到让自己的一些想法能有个简易地实现。所以我想给自己设定一个目标就是坚持100天学习Ruby，每天最少一个番茄时间，并在博客上做记录，开始行动。

最后分享这个坚持100天的行动，希望有更多的人员参与。

参加书法比赛的难题

​白天接到乐妈的电话，说碰到了难题，该怎么办？

起因是乐乐参加了书法班，之前写的一幅作品入选了，要我们报名参加“书法报第九届全国少儿现场书画大赛”桂林赛区的比赛，入选本是好事，只是现在的比赛实在也很难分清东西，而且这比赛在桂林，安排的赛程来回是四天，费用不菲，孩子参赛需要近3千，还不包括来回路费，家长陪同又要差不多2000，实在是难以承受。

 

确实怎么办呢？

报吧，价格这么贵！再说又为什么要去参加呢？本身学习书法也从没想过要参加什么比赛，小乐也纯属书法票友，每周都没写几回，每次要报培训班的时候我们还老打主意让她书法就别报了。

不报吧，怎么和小乐说呢？小家伙听到这个消息肯定是想去的，孩子获得肯定，以及有这样的机会总会希望能试试。

 

乐妈说：“我们直接就和老师说不参加了。”

我坚决说：“不行，必须征求乐乐的意见再定。”

“那我们就不告诉乐乐。”

“那也不行。”

“那怎么和乐乐说呢？”

“我也不知道。”

 

和孩子谈这样的问题觉得好难，让家长有一种歉疚感。会想：“如果就在杭州举办会怎么样？”“如果报名参赛费用不贵会怎么样？”

不过坚定自己的信念，第一我们学习书法并不是为了比赛，就像练钢琴我们也从没想过考级；第二乐乐在书法的练习上投入真的是不够的，说实话也真没到参赛的程度；第三对于这样的比赛我也不认可，回家看到比赛的说明“每组入围全国现场总决赛的选手均可获奖，每组按1:2:1的比例设金奖、银奖、铜奖三个等级，并评出2013年度全国十佳少儿书画之星。”，就更坚定了我的信心。

所以接下来的问题是我怎么和乐乐聊这个话题。

 

想了很多，怎么开头？中间该怎么说？乐乐会怎么表现？执意要去该怎么办？

不知道各位家长是怎么想的，如果你碰到类似情况你会怎么做呢？

镇压？说比赛没意思？晓之以情，动之以理？

 

好吧，继续说晚上我和小乐的对话吧。（乐乐白天已经知道了参赛的事情，并且和乐妈表示想参加。）

晚上在外面散步，乐妈先开的头：“乐乐，书法比赛的事你是不是要和爸爸谈谈？”

“嗯，爸爸，我想去参加比赛的。”

“上次书法比赛你入围了！这么棒的，那肯定很高兴了。”

“是的，就是上次我和zhou在学校里一起写的。”

乐妈插了一句：“不是那个，是你在蒋老师那里写的。”

“哦，那我搞错了，不过我还是想参加的。”

“很想去参加比赛。”

“嗯。”

“你知道比赛是什么样的吗？”

“我不知道，我想先玩了。”乐乐想溜冰了。

“好的，你先玩吧。”

中断了好久，让小家伙先玩过瘾，溜冰结束后继续。

我：“去比赛是不是可以去玩了？”

乐：“是的，是去桂林哎。”

我：“可惜我不能去。”

乐：“爸爸，你为什么不能去啊？”

我：“爸爸还要上班啊。”

乐：“能不能请假的？”

我：“不行啊，8月份我们还要去马来西亚，请不了那么多了。想不想和爸爸一起出去玩的？”

乐：“想的。”

我：“那要不到时我们自己去玩吧。”

乐：“好的，那还有比赛呀。”

我：“是不是所有的比赛我们都要参加的？”

乐：“不是的。”

我：“这次的比赛很远的。”

乐：“桂林在哪里的？”

我：“在广西，咯咯鸡的肚子下面。”

乐：“我知道了，那是很远的。”

我：“是呀，还要好多天。你还有其他课要上，要不这次我们就不去了吧？”

乐：“好的。”

 

就这样，完了。

乐妈刚问我：“你怎么这么容易就说服了呢？”

说容易很容易，说不容易也不容易，我是想了挺久了，但最后在和小乐聊的时候其实我也真不知该怎么说，而真正帮助我达到的还是《如何说孩子才会听，怎么听孩子才肯说》中说到的。

首先是接纳，认可去比赛是高兴的，也是想去的；接着就是耐心，再怎么急，小家伙玩更重要，别急着就想把事情和她一五一十地说完搞定；最后在关键的谈话中别说教，你只需要描述你所看到的，“爸爸上班不能和你们一起去”，“去桂林比赛很远”，“要很多天，你还有其他课要上”，我只是在和小乐聊天的过程中描述事情。

而最后的结果，说实话，也给我带来了惊喜。（很多时候大人想得太复杂，而忽视了事情的本身，使得谈话也受到了误导。）

 

摘录“鼓励孩子与我们合作的五个技巧”：

  1. 描述：描述你所看见的，或者描述问题。

  2. 提示。

  3. 用简单的词语表达。

  4. 说出你的感受。

  5. 写便条。

相同十位数乘法的简便运算

前段时间小乐在背30以内相同两位数的乘法口诀，从11x11, 12x12, 13x13一直到30x30，一开始小乐问我有什么方法的吗？
我说没有，不过老爸会背20以内的，这好像也是小的时候数学老师让死记硬背的，至今也印象深刻。
小乐于是也就没二话了，自己开背，我间或就插问她几个，或者象记3.14一样用趣味联想的方法，比如23x23=529, 27x27=729，在我们的记忆中就成了五阿舅，七阿舅了。

不过昨天在朋友圈中看到一位朋友分享的一个十几乘以十几的乘法方法，据说印度都是这么教的，方法是：
1. 被乘数+乘数的个位数
2. 相加的和乘以10
3. 相乘的积+两个个位数相乘
比如14x16：
1. 14 + 6 = 20
2. 20 x 10 = 200
3. 200 + 4 x 6 = 224
这样基本上直接口算就可以了，而且也超出了原先小乐所背诵的相同的两个数相乘的限制。
再简单证明一下，十几乘以十几用算术表示：
(10 + a) x (10 + b) = 10 x 10 + 10 x a + 10 x b + a x b = (10 + a + b) x 10 + a x b

晚上就和小乐讲解了一下，不过才开始讲，她就拿起书本了，心不在焉，我一看只好先暂停，和她约定一会再讲。

等小乐看书完毕，利用她洗澡刷牙的时间（这也算零碎时间吧）和她说，方法很简单，并先从她熟悉的例子开始举起，比如15x15。
小乐立即能算出225，再进行验证：
1. 15 + 5 = 20
2. 20 x 10 = 200
3. 200 + 5 x 5 = 225
试了几个，小家伙发现果然，就有点来劲了，问：“那二十几乘以二十几呢？”
这也难不到我，我们只需要把第二步的乘以10改成乘以20就可以了，同样用小乐熟悉的25x25验证：
1. 25 + 5 = 30
2. 30 x 20 = 600
3. 600 + 5 x 5 = 625
是不是很简单啊？于是这次变成了我问：“那三十几乘以乘以三十几呢？”
“乘以三十。”
“对了。”
“那看来以后我做乘法都可以很简单地口算了。”
“不过可要注意，这里的乘法方法是要求两个数字的十位数是相同的，要是15x25可就不一样了。”我赶紧提醒。

零碎时间创造奇迹

乐妈要参加家长会，今晚我当差，陪小乐去上英语课。陪其实也只是陪而已，上课的内容、作业、问题等等我都是不关心，这自然由小乐自己负责搞定。

上英语课的林老师年纪挺大，是我们全家都很欣赏的一位老师，不仅仅是英语的水平，尤其是其为人和好学的精神，近70岁的人仍然好学不倦，对电脑教学也很有兴趣，之前有一次就自己一个劲地研究怎么转换音频，折腾出来后感觉很有成就感，实在是非常让人敬佩。

林老师的英语课非常重视发音，以前小乐学习回来，就和我说：“知道怎么念英语吗？要注意语音，语调，意群，连读，失去爆破，情感表达，节奏。”当时就把我给震懵了，都不敢在小乐面前念英语了。也好这样我就找到借口放手不管了;-)。今天听课，就又感受了一回。小朋友的朗读都很不错，每个小朋友朗读或背诵时，其他小朋友可以抓虫，挑出其中的毛病，学中带着玩，也挺有意思。

临下课的时候，林老师介绍了李阳的英语学习方法，其中提到了“零碎时间创造奇迹”。

因为不管多忙碌，一个人一天可以找出至少二十多个“五分钟”的零碎时间！而五分钟可以干什么呢？一分钟可以把一个句子操练二十遍，五分钟就可以掌握五个句子；五分钟可以把一篇对话操练十至十五遍；五分钟可以把一篇短文操练五至十遍！

零碎时间从哪里来？等公共汽车、生活中永远不可制止的排队、一日三餐的前后、人在旅途（包括上下学的路上）的时光，都是学习英语的好时机。甚至上厕所的时候，也不妨背上一个句子！我特别喜欢堵车，因为一堵车，我别无选择，只有拿出小纸片来背上两句。我特别喜欢排队，因为再长的队，我都没有感觉，好像一会儿就轮到我了。
不知道乐妈是不是看过或听别人介绍过这个方法，小乐的英语学习还真有点体现这个零碎时间学习的思路。我们尽量想让她能有时间玩，而那么多需要学习的内容怎么办呢？关键一点就是要“把学习化整为零，而把玩的时间化零为整”。（看书对于小乐应该属于和玩一个级别的）一定要让孩子感觉到是有足够的时间玩的。

那么零碎时间在哪里呢？上学路上乐妈会陪她背背单词，出门车上就放着英语故事，在家中更是如此，无论是吃饭、睡觉、看书做作业期间，都经常是循环播放着某一个有趣的英语故事（比如伊索寓言或者英文儿歌），缔造一个语言环境的氛围吧，毕竟我和乐妈都是属于英语开不了口的那种，也只有让她自己去体会“语音，语调，意群，连读，失去爆破，情感表达，节奏”了。

好在乐乐在语言上有一定的领悟能力，在英语学习中往往也能注入自己的情感，在这一点上我觉得小时候有一点起到挺好的作用，就是家长有感情地讲故事，并且投入到角色扮演中去。记得从乐乐记事开始，我们全家就都是属于疯玩的那种，几乎每一部看过的动画片或者故事，我们都是要带演的，所以小乐从一开始自己讲故事的时候，就是声情并茂的，这应该是一个很好的基础，也带入了英语学习中。

真诚

“那么结束谎言吧，让我们彼此讲真话，因为我们彼此相互属于，就像同一个身体的各个部分。” —— 《以弗所书》，4：25
这周的家庭美德是真诚。
“真诚，意味着你的言行都是真实的。说真话就是哪怕为了保护自己或他人也不会撒谎。”

“我觉得有问题。”我先插了一句，“不是说对坏人可以撒谎的嘛！”
“是呀。”小乐和乐妈都表示同意。
记得我们在讨论“诚实”的美德时，就谈论过这个问题，当时就因为对日常发生的一些坏事的担心，比如上学路上有人拐走孩子，所以趁机教育小乐对坏人应该可以不说真话。
那对这句“说真话就是哪怕为了保护自己或他人也不会撒谎”该怎么理解呢？
我说：“也许对坏人说谎也是很真诚的吧？”
“说谎还怎么真诚呀？”小乐不理解。
我也被难住了。
乐妈这时举了个例子：“前两天的报纸上报道，一个十多岁的姐姐进门的时候，被坏人跟进了房间。警察后来就建议孩子可以故意说‘你找爸爸吗，他在房间里睡觉’什么的。”
小乐急切地问：“后来怎么样了？”
“这个姐姐倒是很真诚地说别伤害自己，可以把压岁钱都给他，后来那个坏人拿了钱就走了。”

说实话，每次看到这样的新闻，心情都很糟，觉得缺少一种安全感，在这个时候只有对孩子再进行一次安全教育，要注意保护自己。

继续聊我们的“真诚”。
乐妈说：“我昨天倒是很真诚的。”
“什么？”
“昨天简简弟弟要睡到我们的床上，我就告诉他我有洁癖，裤子在地上坐过就不能再上床了。虽然很难为情，但我还是说了真话。”
乐乐点点头：“简简弟弟也很真诚地，他问什么是洁癖呀？”
我听得哈哈直笑：“不过确实把自己真实的想法给说出来反而更踏实，不然就要靠撒谎应付了。”
“怎么撒谎呀？”
“比如告诉简简弟弟今天自己去睡，就给他一百个游戏玩。”

“有人告诉你你最好的朋友说了你的坏话，并且再也不想和你做朋友了……”
乐妈想举个例子，A、B、C啥的，越说越难说明白。
我说：“A friend in need is a friend indeed，有些口口声声说是你朋友的人不一定是你真正的朋友.” （这是乐乐最近听的伊索寓言中的故事）
乐乐说：“那这个也有问题，最好的朋友怎么会说你的坏话呢？”
“这里就是有人在说，但需要你去判断，他说的是不是一定是真实的？如果你知道之后，你会怎么办呢？”
小乐回答：“我会直接去问问我的好朋友。”
“Good job!”
乐妈：”好朋友之间是应该可以坦诚相见的。”

“你发现自己夸大了自己在一个比赛中的成绩……”
“这个上次在你的作文中没有夸大你的跑步成绩，实事求是地说了有两个同学超过了你。”乐妈说。
“是的。”
我说：“那也是真诚的表现。”
乐妈：“现在你应该更能明白虚假和虚拟的区别了吧，你可以用真诚来做判断。”
我们以前讨论过关于写作文是否可以夸大，也谈论过以前我们自己为了考试就背牢别人的作文来应付去得高分，这都是应试教育的结果，而之前小乐也面临这样的困惑，我们的建议是可以虚拟，但不要虚假。

生吞活剥

早上陪小乐去上篮球课，走在路上的时候说起了“生吞活剥”。三毛的文章中频繁地出现这个成语：

第一次看到是在“逃学为读书”，“我的看书，在当时完全是生吞活剥，无论真懂假懂，只要故事在，就看得下去，有时看到一段好文章，心中也会产生一丝说不出的滋味来，可是我不知道那个字原来叫做‘感动’。”

之后“蓦然回首”中，“在那几天生吞活剥的急切求知里，我将自己累得虚脱，而我的心，我的欢喜，我的兴奋，是胀饱了的帆船——原来我不寂寞，世上有那么多似曾相识的灵魂啊！”

而在“补考定终生”中，“当时对于哲学，兴趣很浓，手边放着康德的《纯理性批判》为主书，其他尚有一百本以上形形色色哲学副书，都在生吞活剥。”

我说：“三毛是不是江郎才尽，找不到更好的词了。”
“不会吧。”
“那你说生吞活剥是什么意思呢？”
小乐想了想，说：“我用一个词组来说好不好？”
“好的。”
“贪婪。”
“有道理，不过感觉还差点什么，有什么意思没能表示出来。”
“那你说用什么？”小乐反问。
“我觉得急不可待，很急切的心情。”
“那也可以迫不及待，狼吞虎咽。”
“不过，这个就又只有急了，又少了其他的味道。”
“哎，还真不好找啊！”

到底生吞活剥是什么意思呢？百度百科中解释：

“生吞活剥”出自南宋计有功的《唐诗纪事》。唐朝初年，枣强县尉张怀庆喜欢抄袭著名文人的文章。当朝大臣李义府曾写了一首五言诗，原文是：“镂月成歌扇，裁云作舞衣，自怜回雪影，好取洛川归。”张怀庆将这首诗改头换面，在每句的前头加上两个字，变成一首七言诗：“生情镂月成歌扇，出性裁云作舞衣：照镜自怜回雪影，来时好取洛川归。”人们读了张怀庆的这首诗，无不哗然大笑。有人讥讽他这种手段是：“活剥王昌龄，生吞郭正一！”王、郭都是当时以文词闻名的朝中要人，唐高宗的诏书和朝廷文告，多半出自他们的手笔。

后来，人们根据这个故事引申出“生吞活剥”这句成语，比喻生搬硬套，或剽窃照抄别人现成的词句、理论、经验等等，而不知融会贯通。现在多用来比喻学了别人的东西不能消化吸收，只知生硬地接受或机械地搬用。
原来我和小乐的理解都还是有问题，没能把不消化吸收的意思给表示出来。
这个出处也很有意思，我还一直以为“生吞活剥”应该是出自吃东西上，类似狼吞虎咽之类的。问小乐觉得这个词的出处，小家伙说：“原始人吃东西。”和我一样。

不过在清袁枚的《随园诗话》卷一中：“熊掌豹胎，食之至珍贵者也；生吞活剥，不如一蔬一笋矣。”倒确实有不加烹调之意，倒是也吃挂钩了。

不要急着给孩子提建议

昨天发的旧文“理解孩子不是件容易的事”没想到得到了家长朋友的共鸣，得到以前的同事fire随喜和maggie的推荐，今天涨了不少粉丝，对我这样自娱自乐的确实是不少了，要谢谢两位。

于是想继续对昨天的话题讨论下，其实这样的事情每天都会遇到，我们一家人也在每次的经历中成长。刚好看到了乐妈今天的日记，和大家一起分享一下：

晚上，接到同学妈妈的电话，问小主持报名的事。小乐忘了告诉我了，我看了报名表，如果报名后，以后就要周日上课的，于是就直接对小乐说不报了吧。但马上发现自己又犯错了，这事应该小乐自己来决定的。
“直接对小乐说不报”，这是否定式回答，当然我们现在都意识这个效果很差，只不过偶尔还是会因为家长意识，很轻易地冒出来。我认为意识到否定式回答的问题，家长们就进入第二境界了，接下来是练习，强化自己的第一反应，大家可以尝试更好的回答。

在《如何说孩子才会听，如何听孩子才肯说》中提到帮助孩子面对她们的感受时，提到了四种方法：
1. 安静专心地倾听。
2. 用简单的词语回应她们的感受。“哦….嗯…原来是这样…”
3. 说出她们的感受。“这件事让你很灰心…”
4. 用幻想的方式实现她们的愿望。“真希望我是齐天大圣，给你把香蕉变出来。”

第一次否定回答后，乐妈尝试要做弥补，所以又和小乐做了交流。

等她弹好琴，我带她下去玩轮滑。问她是怎么想这个报名的事的，她说：“他们都说我有主持的能力，但是我自己不这么想，我并不想以后做主持，我想当一位思想家、教育家、还有至少拿一项诺贝尔奖的人！” （呵呵，小家伙的野心不小，就是没有实现的行动！）

“那你报这么多兴趣班开心吗？”我一边牵着她的手，帮助她保持平衡。

“开心，这些都是我喜欢的，妈妈，你别看我有时弹琴好像不开心，其实我是不喜欢有的曲子，像这次的这首曲子太难听了！”

也是，这周的练习是半音阶练习，听起来不优美，“那你可以自己和紫衣老师说，能不能换一首好听点又能达到练习效果的曲子呀！”
这样的交流对她们俩来说是很平常的，两个人能开开心心地聊。不过我刚才和乐妈说：“你觉得你说的有什么问题吗？”
（建议家长朋友可以在这里再回过去看看，并想想自己会怎么说）
“没有啊，我还想等你表扬呢！”
“是要表扬的，不过也许最后一句话可以换成你心里所想的那句话。”

就是把心里想的“也是，这周的练习是半音阶练习，听起来不优美。”给说出来，回应小乐。
这句话和“那你可以自己和紫衣老师说，能不能换一首好听点又能达到练习效果的曲子呀！”的区别就在于，前者是上述方法中的说出小乐心中的感受，而后者则是直接给出了建议。

建议有什么不好呢？它剥夺了孩子解决问题的过程和乐趣，使得孩子会越来越产生依赖。这又是我们经常会犯的一个错误，急着就想告诉应该怎么样，不应该怎么样。我们家里提建议最多的是乐乐外婆（还好，老太太不看我的微信），不管是吃饭、睡觉、洗碗，总想时刻告诉你们最佳实践，并监督执行，而这恰恰有时会引起反感，就像孩子有时不喜欢被问问题一样，她也不喜欢被监督。

我自己在给小乐讲数学作业时，都是一步一步迈进，从来不说多的，如果她说明白了，我就到此为止，即时有时对奥数题还是一知半解。抑或她做数学作业的时候，看着她口算没出来，即使速度再慢，我也不会催着她，或告诉她答案，而是耐心等她，慢慢来。

这也是需要我们不断地练习的。
